### Issues Found (2)

**1. reset-password not rate-limited (AUTH-02, AUTH-03)**
- `src/middleware.ts` line 165 — `publicApiRateLimiter` covers `forgot-password` and `set-password` but NOT `reset-password`
- Risk: Token guessing attacks on the reset endpoint are unlimited
- Fix: Add `|| pathname.startsWith('/api/auth/reset-password')` to the rate limiter condition

**2. Signup flow bypasses httpOnly cookie auth (AUTH-04, AUTH-05)**
- `src/app/api/organizations/signup/route.ts` line 87 — returns JWT in response body only, no httpOnly cookie set
- `src/app/signup/page.tsx` lines 73-83 — writes JWT to localStorage
- Risk: New orgs start in legacy auth mode; CSRF protection never activated for signup sessions
- Fix: Set `auth-token` and `csrf-token` cookies on signup response (matching login route pattern); remove localStorage writes in signup page
