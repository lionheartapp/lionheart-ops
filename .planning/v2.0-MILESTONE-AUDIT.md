## Integration Check Results

**Missing connections:** 2

### Missing Connection 1: Billing Permission Assignment (SET-02)

`PERMISSIONS.SETTINGS_BILLING` is defined in `permissions.ts` line 47 and enforced in all 4 billing API routes via `assertCan`. However, it is NOT assigned to `DEFAULT_ROLES.ADMIN`. Only `super-admin` (wildcard `*:*`) can access billing. Admin-role users see the Billing tab in the UI but every API call returns 403.

**Fix:** Add `PERMISSIONS.SETTINGS_BILLING` to `DEFAULT_ROLES.ADMIN.permissions` in `src/lib/permissions.ts`.

### Missing Connection 2: Observability Gap (INFRA-03, INFRA-04)

Phase 13 Plan 02 migrated 30 routes to Pino/Sentry but was completed before Phases 10-15 added 21 new routes. These routes lack structured logging and error tracking.
