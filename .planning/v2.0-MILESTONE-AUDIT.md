## Integration Findings (3 items, 0 blockers)

### INT-01 (P1): Bulk Notifications Bypass Preferences

`createBulkNotifications` in `notificationService.ts:119` calls `prisma.notification.createMany` directly, skipping `pauseAllNotifications` and `inAppEnabled` checks. Users who disable notification types still receive them via the bulk path (inventory, maintenance, IT, compliance).

**Fix:** Filter recipients through preference lookup before `createMany`, or route through `createNotification` individually.

### INT-02 (P2): AI Routes Missing Observability

3 AI routes (`chat`, `generate-description`, `parse-event`) lack Pino and Sentry instrumentation. Created before Phase 13 and omitted from Phase 16 retrofit.

**Fix:** Add `logger.child` and `Sentry.captureException` to these 3 routes.

### INT-03 (P3): Resend-Verification Rate Limiting

`POST /api/auth/resend-verification` is public but not in the middleware `publicApiRateLimiter` branch. Has application-level 3/hour throttle via DB token count.

**Fix:** Add to `publicApiRateLimiter` branch in middleware for consistency.
