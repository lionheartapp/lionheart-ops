**Step 1: Create src/lib/types/maintenance-ai.ts**
Define the typed cache structure for `MaintenanceTicket.aiAnalysis` (Json? field):
```typescript
export interface AiDiagnosis {
  likelyDiagnosis: string
  confidence: 'LOW' | 'MEDIUM' | 'HIGH'
  confidenceReason: string
  suggestedTools: string[]
  suggestedParts: string[]
  steps: string[]
  analyzedPhotoCount: number
  analyzedAt: string  // ISO timestamp
}

export interface AiConversationTurn {
  role: 'user' | 'assistant'
  content: string
  timestamp: string
}

export interface AiAnalysisCache {
  diagnosis: AiDiagnosis | null
  conversation: AiConversationTurn[]
  lastPhotoSnapshot: string[]  // Copy of photos[] at time of analysis — for invalidation check
}
```

**Step 2: Create src/lib/services/ai/maintenance-ai.service.ts**
Anthropic SDK wrapper service:
- Import `Anthropic` from `@anthropic-ai/sdk`
- Instantiate client: `new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })`
- Use model `claude-sonnet-4-5-20250929` per project decision in STATE.md (Anthropic Claude, not Gemini)

**`analyzeMaintenancePhotos(params)`:**
- Accept: `{ photoUrls: string[], category: string, title: string, description?: string | null }`
- Build message content array: one `{ type: 'image', source: { type: 'url', url } }` per photo URL (Supabase HTTPS URLs work directly), plus one `{ type: 'text', text: DIAGNOSTIC_PROMPT }`
- DIAGNOSTIC_PROMPT should instruct the model to analyze maintenance photos and return JSON with: likelyDiagnosis (string), confidence (LOW/MEDIUM/HIGH), confidenceReason (string), suggestedTools (string[]), suggestedParts (string[]), steps (string[]). Include the ticket category and title/description for context. Instruct to return ONLY valid JSON, no markdown wrapping.
- Call `client.messages.create({ model, max_tokens: 1024, messages: [{ role: 'user', content }] })`
- Parse the text response as JSON, validate structure, return as `AiDiagnosis` (add `analyzedPhotoCount` and `analyzedAt`)
- Wrap in try/catch — on any error (parse failure, API error), return null

**`askMaintenanceAI(params)`:**
- Accept: `{ question: string, ticketContext: { category: string, title: string, description?: string | null, photos: string[] }, conversationHistory: AiConversationTurn[] }`
- Build messages array with system context about the ticket + previous conversation turns + new user question
- Photos included as URL sources only if present
- Call `client.messages.create({ model, max_tokens: 1024, messages })`
- Return the assistant's text response as string, or null on error

**Step 3: Create POST /api/maintenance/tickets/[id]/ai-diagnose/route.ts**
- Follow standard route pattern: `getOrgIdFromRequest` -> `getUserContext` -> `assertCan(ctx.userId, PERMISSIONS.MAINTENANCE_CLAIM)` (technicians and heads can use AI) -> `runWithOrgContext`
- Graceful degrade: if `!process.env.ANTHROPIC_API_KEY`, return `ok({ available: false, diagnosis: null, cached: false })`
- Fetch the ticket by ID (include photos, aiAnalysis, category, title, description)
- If no photos on ticket, return `ok({ available: true, diagnosis: null, cached: false, reason: 'no-photos' })`
- Check cache: cast `ticket.aiAnalysis as AiAnalysisCache | null`. If `cached?.diagnosis` exists AND `JSON.stringify(ticket.photos.sort()) === JSON.stringify((cached.lastPhotoSnapshot ?? []).sort())`, return cached diagnosis with `cached: true`
- Otherwise: call `analyzeMaintenancePhotos(...)` from the service
- If diagnosis returned: build updated `AiAnalysisCache` (preserve existing conversation turns), write to `prisma.maintenanceTicket.update({ where: { id }, data: { aiAnalysis: updatedCache as any } })`
- Return `ok({ available: true, diagnosis, cached: false })`

**Step 4: Create POST /api/maintenance/tickets/[id]/ai-ask/route.ts**
- Same auth pattern as ai-diagnose
- Graceful degrade for missing API key
- Validate request body: `{ question: string }` with Zod (min 1 char, max 1000 chars)
- Fetch ticket (include photos, aiAnalysis, category, title, description)
- Load existing conversation from `ticket.aiAnalysis as AiAnalysisCache | null`
- Call `askMaintenanceAI(...)` with the question, ticket context, and existing conversation
- If response returned: append user turn + assistant turn to `cache.conversation`, write updated cache to DB
- Return `ok({ answer: responseText, conversation: updatedConversation })`
- If AI call fails: return `ok({ answer: null, error: 'AI service unavailable' })`
