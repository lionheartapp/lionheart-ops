**Step 5: Create KanbanBoard.tsx**
Main board component wrapping everything:
- `DndContext` with `MouseSensor` (activationConstraint: { distance: 8 }) and `TouchSensor` (activationConstraint: { delay: 200, tolerance: 5 })
- `closestCorners` collision detection
- 6 columns: BACKLOG, TODO, IN_PROGRESS, ON_HOLD, QA, DONE (const `BOARD_COLUMNS`)
- SCHEDULED and CANCELLED are NOT shown as columns
- `DragOverlay` renders KanbanCard with `isOverlay` prop for visual treatment
- Props: tickets (WorkOrderTicket[]), filters, view mode (my-board/campus/all), permissions, onFilterChange
- Group tickets by status into column buckets

**onDragStart:** Set `activeTicket` state for DragOverlay rendering.

**onDragOver:** Track which column is being hovered for valid/invalid highlighting. Compute validity from `ALLOWED_TRANSITIONS[activeTicket.status][overColumnStatus]`.

**onDragEnd handler:**
1. If dropped outside (no `over`): clear state, return
2. If dropped on a tech avatar (over.id starts with `tech-`): fire assign mutation, return
3. Get `newStatus` from `over.id`
4. If `!ALLOWED_TRANSITIONS[ticket.status]?.[newStatus]`: trigger shake animation on card + `toast.error("Cannot move from X to Y")`, return
5. If `newStatus === 'ON_HOLD'`: set `pendingDrop = { ticketId, newStatus }`, show HoldReasonInlineForm in a modal wrapper (NOT inline — it's a drag context), on confirm fire status PATCH with holdReason + holdNote, on cancel clear pendingDrop
6. If `newStatus === 'QA'`: set `pendingDrop = { ticketId, newStatus }`, show QACompletionModal, on confirm fire status PATCH with completionNote + completionPhotos, on cancel clear pendingDrop
7. For gated transitions (ON_HOLD, QA): do NOT move card optimistically before modal confirms. Show card in original column with a subtle "pending" visual indicator.
8. For non-gated valid transitions: fire optimistic mutation immediately using TanStack Query pattern: `onMutate` snapshots cache and moves card in local data, `onError` rolls back, `onSettled` invalidates
