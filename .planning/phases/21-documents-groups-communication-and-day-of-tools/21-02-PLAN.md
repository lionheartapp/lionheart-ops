<task type="auto">
  <name>Task 1: Create document and compliance service layer</name>
  <files>src/lib/services/eventDocumentService.ts</files>
  <action>
Create `src/lib/services/eventDocumentService.ts` with the following functions:

**Document Requirements:**
- `createDocumentRequirement(data: { eventProjectId, label, description?, documentType, isRequired?, dueDate?, sortOrder? })` — creates an EventDocumentRequirement. Auto-creates empty EventDocumentCompletion rows for all existing REGISTERED participants (so the matrix view has rows immediately). Uses `prisma` (org-scoped).
- `updateDocumentRequirement(id: string, data: Partial<...>)` — update label, description, type, dueDate, isRequired, sortOrder.
- `deleteDocumentRequirement(id: string)` — hard-delete the requirement and cascade-deletes completions.
- `listDocumentRequirements(eventProjectId: string)` — returns all requirements for an event, sorted by sortOrder.

**Completion Matrix:**
- `getDocumentMatrix(eventProjectId: string)` — returns the matrix data structure: array of participants (registrationId, firstName, lastName, photoUrl) each with an array of requirement completions (requirementId, isComplete, completedAt, fileUrl). Queries EventRegistration (status=REGISTERED) joined with EventDocumentCompletion. Missing completions treated as incomplete.
- `toggleCompletion(registrationId: string, requirementId: string, isComplete: boolean)` — upserts an EventDocumentCompletion record. If marking complete, sets completedAt to now(). If marking incomplete, clears completedAt.

**Reminders:**
- `sendDocumentReminder(eventProjectId: string, options: { requirementId?: string })` — finds all registrations with incomplete documents (or incomplete for a specific requirement). Calls registrationEmailService or a simple email function to send a reminder email to each registration's email address. Returns count of reminders sent.
- `getIncompleteParticipants(eventProjectId: string)` — returns list of registrations with at least one incomplete required document.

**Compliance Checklist:**
- `listComplianceItems(eventProjectId: string)` — returns all compliance items sorted by sortOrder.
- `upsertComplianceItem(data: { id?, eventProjectId, label, description?, status?, assigneeId?, dueDate?, fileUrl?, sortOrder? })` — create or update a compliance item.
- `deleteComplianceItem(id: string)` — hard-delete.
- `getDefaultComplianceChecklist()` — returns a static array of common off-campus compliance items (liability insurance, vehicle inspection, driver background checks, vendor contracts, venue safety cert, emergency action plan) that staff can import.

Use `prisma` (org-scoped) for all queries inside routes that call runWithOrgContext. Import from `@/lib/db`.

  <files>src/app/api/events/projects/[id]/documents/route.ts, src/app/api/events/projects/[id]/documents/completions/route.ts, src/app/api/events/projects/[id]/documents/reminders/route.ts, src/app/api/events/projects/[id]/compliance/route.ts</files>
  <action>
Create 4 API route files following the established route pattern: getOrgIdFromRequest → getUserContext → assertCan → runWithOrgContext → service call → ok()/fail() response.

**1. `/api/events/projects/[id]/documents/route.ts`:**
- GET: assertCan(EVENTS_DOCUMENTS_MANAGE), returns listDocumentRequirements(id). Also returns basic stats: total requirements, total registrations, completion percentage.
- POST: assertCan(EVENTS_DOCUMENTS_MANAGE), validates with Zod (label required, documentType required from enum), calls createDocumentRequirement. Returns 201.
- PUT: assertCan(EVENTS_DOCUMENTS_MANAGE), validates with Zod, calls updateDocumentRequirement. Requires `requirementId` in body.
- DELETE: assertCan(EVENTS_DOCUMENTS_MANAGE), requires `requirementId` in body (or query param), calls deleteDocumentRequirement.

**2. `/api/events/projects/[id]/documents/completions/route.ts`:**
- GET: assertCan(EVENTS_DOCUMENTS_MANAGE), returns getDocumentMatrix(id). Response shape: `{ requirements: [...], participants: [{ registration, completions: [...] }] }`.
- PATCH: assertCan(EVENTS_DOCUMENTS_MANAGE), validates `{ registrationId, requirementId, isComplete }` with Zod, calls toggleCompletion.

**3. `/api/events/projects/[id]/documents/reminders/route.ts`:**
- POST: assertCan(EVENTS_DOCUMENTS_MANAGE), optional `{ requirementId }` in body, calls sendDocumentReminder. Returns `{ sent: number }`.

**4. `/api/events/projects/[id]/compliance/route.ts`:**
- GET: assertCan(EVENTS_COMPLIANCE_MANAGE), returns listComplianceItems(id). Optionally accepts `?defaults=true` to return getDefaultComplianceChecklist() instead.
- POST: assertCan(EVENTS_COMPLIANCE_MANAGE), validates with Zod, calls upsertComplianceItem. Returns 201.
- PUT: assertCan(EVENTS_COMPLIANCE_MANAGE), validates with Zod, calls upsertComplianceItem (update path).
- DELETE: assertCan(EVENTS_COMPLIANCE_MANAGE), requires `itemId`, calls deleteComplianceItem.

All routes: Zod validation on inputs, proper error handling with fail() for 400/403/500, use ok() envelope for success responses.
