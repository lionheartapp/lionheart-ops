**src/lib/offline/event-sync.ts:**
Sync module for event offline data:

- `syncCheckIns(eventProjectId: string)` — reads all pending check-ins from eventCheckInQueue, PUTs to /api/events/projects/${id}/check-in (sync endpoint). On success, marks as 'synced'. On failure, marks as 'failed' with error. Returns { synced, failed }.
- `syncIncidents(eventProjectId: string)` — reads all pending incidents from eventIncidentQueue, PUTs to /api/events/projects/${id}/incidents (sync endpoint). Same pattern.
- `cacheRoster(eventProjectId: string)` — fetches GET /api/events/projects/${id}/check-in?full=true and stores all participants in cachedParticipants. Called when online to pre-populate the offline cache.
- `getCachedParticipant(registrationId: string)` — looks up a participant from the local cache. Returns participant data or null.
- `syncAll(eventProjectId: string)` — calls syncCheckIns + syncIncidents. Intended to be called on connectivity restore.
- `getPendingCount()` — returns count of pending (unsynced) check-ins and incidents.

Use navigator.onLine to detect connectivity. Listen for 'online' event to trigger syncAll automatically.

**src/lib/hooks/useCheckIn.ts:**
- `useCheckIn(eventProjectId: string)` — hook that manages check-in with online/offline support:
  - `checkIn(registrationId: string, method: 'QR_SCAN' | 'MANUAL')`:
    - If online: POST to API, return result.
    - If offline: write to Dexie eventCheckInQueue, look up participant from cachedParticipants, return cached info.
  - `counter` — GET /api/events/projects/${id}/check-in counter. Poll every 5s when online. When offline, derive from cached data + pending queue.
  - `checkInList` — full status list. Poll every 10s when online.
  - `undoCheckIn(registrationId: string)` — DELETE to API (online only; offline undo removes from queue).
  - `isOnline` — boolean, updates on connectivity change.
  - `pendingSync` — count of unsynced items.
  - `syncNow()` — manually trigger sync.

**src/lib/hooks/useIncidents.ts:**
- `useIncidents(eventProjectId: string)` — hook with online/offline support:
  - `createIncident(data)`:
    - If online: POST to API.
    - If offline: write to Dexie eventIncidentQueue.
  - `incidents` — list from API (poll every 30s when online) merged with pending offline incidents.
  - `pendingSync` — count of unsynced incidents.

Keep all existing maintenance caching rules. Add the new rules at the end of the runtimeCaching array.
