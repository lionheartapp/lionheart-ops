<task type="auto">
  <name>Task 1: Add all Phase 21 Prisma models and enums</name>
  <files>prisma/schema.prisma</files>
  <action>
Add the following models and enums to schema.prisma. Follow the established pattern: cuid() for IDs, organizationId FK, proper relations, cascade rules. Reference existing EventProject and EventRegistration models.

**New Enums:**
- `EventGroupType` — BUS, CABIN, SMALL_GROUP, ACTIVITY
- `ComplianceItemStatus` — NOT_STARTED, IN_PROGRESS, COMPLETE
- `IncidentSeverity` — MINOR, MODERATE, SERIOUS
- `IncidentType` — MEDICAL, BEHAVIORAL, SAFETY, OTHER
- `AnnouncementAudience` — ALL, GROUP, INCOMPLETE_DOCS, PAID_ONLY
- `SurveyStatus` — DRAFT, ACTIVE, CLOSED

**New Models (all org-scoped):**

1. `EventDocumentRequirement` — Required document definition per event.
   Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), label (String), description (String?), documentType (String — permission_slip, waiver, medical_release, photo_release, custom), isRequired (Boolean, default true), dueDate (DateTime?), sortOrder (Int, default 0), createdAt, updatedAt.
   Add reverse relation `documentRequirements EventDocumentRequirement[]` on EventProject.

2. `EventDocumentCompletion` — Per-participant completion tracking.
   Fields: id, organizationId, eventProjectId (String), registrationId (FK→EventRegistration, Cascade), requirementId (FK→EventDocumentRequirement, Cascade), isComplete (Boolean, default false), completedAt (DateTime?), signatureId (String? — optional FK→RegistrationSignature), fileUrl (String?), notes (String?), createdAt, updatedAt.
   Unique constraint: @@unique([registrationId, requirementId]).
   Index: @@index([eventProjectId, isComplete]).

3. `EventComplianceItem` — Off-campus event compliance checklist.
   Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), label (String), description (String?), status (ComplianceItemStatus, default NOT_STARTED), assigneeId (String?, FK→User, SetNull), dueDate (DateTime?), fileUrl (String?), sortOrder (Int, default 0), createdAt, updatedAt.
   Add reverse relation `complianceItems EventComplianceItem[]` on EventProject.
   Named relation for assignee: "ComplianceItemAssignee".

4. `EventGroup` — Group definition (bus, cabin, small group, activity).
   Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), name (String), type (EventGroupType), capacity (Int?), leaderId (String?, FK→User, SetNull — named "GroupLeader"), description (String?), sortOrder (Int, default 0), createdAt, updatedAt.
   Add reverse relation `groups EventGroup[]` on EventProject.

5. `EventGroupAssignment` — Participant-to-group mapping.
   Fields: id, organizationId, registrationId (FK→EventRegistration, Cascade), groupId (FK→EventGroup, Cascade), assignedAt (DateTime, default now()), assignedById (String?, FK→User, SetNull — named "GroupAssigner").
   Unique constraint: @@unique([registrationId, groupId]).
   Index: @@index([groupId]).

6. `EventActivityOption` — Elective activity for signup during registration.
   Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), name (String), description (String?), capacity (Int?), scheduledAt (DateTime?), durationMinutes (Int?), locationText (String?), sortOrder (Int, default 0), createdAt, updatedAt.
   Add reverse relation `activityOptions EventActivityOption[]` on EventProject.

7. `EventActivitySignup` — Participant signup for an activity.
   Fields: id, organizationId, registrationId (FK→EventRegistration, Cascade), activityId (FK→EventActivityOption, Cascade), signedUpAt (DateTime, default now()).
   Unique constraint: @@unique([registrationId, activityId]).
   Index: @@index([activityId]).

8. `EventAnnouncement` — Targeted announcement.
   Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), title (String), body (String), audience (AnnouncementAudience, default ALL), targetGroupId (String?, FK→EventGroup, SetNull — named "AnnouncementTargetGroup"), sentAt (DateTime?), createdById (String, FK→User, Restrict — named "AnnouncementAuthor"), createdAt, updatedAt.
   Add reverse relation `announcements EventAnnouncement[]` on EventProject.

9. `EventCheckIn` — QR code check-in record.
   Fields: id, organizationId, eventProjectId (String), registrationId (FK→EventRegistration, Cascade), checkedInAt (DateTime, default now()), checkedInById (String?, FK→User, SetNull — named "CheckInStaff"), method (String, default "QR_SCAN" — QR_SCAN or MANUAL), syncedAt (DateTime? — null if created offline), createdAt.
   Unique constraint: @@unique([registrationId, eventProjectId]).
   Index: @@index([eventProjectId, checkedInAt]).

10. `EventIncident` — Day-of incident logging.
    Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), type (IncidentType), severity (IncidentSeverity), description (String), actionsTaken (String?), followUpNeeded (Boolean, default false), followUpNotes (String?), photoUrl (String?), reportedById (String, FK→User, Restrict — named "IncidentReporter"), syncedAt (DateTime?), createdAt, updatedAt.
    Add reverse relation `incidents EventIncident[]` on EventProject.

11. `EventIncidentParticipant` — Junction: incident to involved participants.
    Fields: id, registrationId (FK→EventRegistration, Cascade), incidentId (FK→EventIncident, Cascade).
    Unique constraint: @@unique([registrationId, incidentId]).

12. `EventSurvey` — Post-event feedback survey.
    Fields: id, organizationId, eventProjectId (FK→EventProject, Cascade), formId (String, FK→RegistrationForm, Restrict — reuses form builder), status (SurveyStatus, default DRAFT), opensAt (DateTime?), closesAt (DateTime?), createdAt, updatedAt.
    Add reverse relation `surveys EventSurvey[]` on EventProject.

13. `EventSurveyResponse` — Individual survey submission.
    Fields: id, organizationId, surveyId (FK→EventSurvey, Cascade), registrationId (FK→EventRegistration, Cascade), responses (Json), submittedAt (DateTime, default now()).
    Unique constraint: @@unique([surveyId, registrationId]).

14. `EventPresenceSession` — Real-time presence tracking (for audit; actual presence via Supabase Realtime).
    Fields: id, organizationId, eventProjectId (String), userId (String, FK→User, Cascade — named "PresenceUser"), activeTab (String?), lastSeenAt (DateTime, default now()).
    Unique constraint: @@unique([eventProjectId, userId]).

Also add reverse relations on EventRegistration for: documentCompletions, groupAssignments, activitySignups, checkIns, incidentInvolvements, surveyResponses.

Also add reverse relations on User for: complianceItemsAssigned, groupsLed, groupAssignmentsMade, checkInsPerformed, incidentsReported, announcementsCreated, presenceSessions.

Run `npx prisma validate` to confirm schema is valid.

**src/lib/permissions.ts:**
Add new permission constants to the PERMISSIONS object:
- `EVENTS_DOCUMENTS_MANAGE: 'events:documents:manage'` — create/edit document requirements, send reminders
- `EVENTS_COMPLIANCE_MANAGE: 'events:compliance:manage'` — manage compliance checklist items
- `EVENTS_GROUPS_MANAGE: 'events:groups:manage'` — create groups, assign participants, manage activities
- `EVENTS_CHECKIN_MANAGE: 'events:checkin:manage'` — scan QR codes, perform check-ins
- `EVENTS_ANNOUNCEMENTS_MANAGE: 'events:announcements:manage'` — post announcements
- `EVENTS_INCIDENTS_MANAGE: 'events:incidents:manage'` — log and view incidents
- `EVENTS_SURVEYS_MANAGE: 'events:surveys:manage'` — create and manage post-event surveys

Add ALL 7 permissions to the ADMIN role in DEFAULT_ROLES.
Add EVENTS_DOCUMENTS_MANAGE, EVENTS_GROUPS_MANAGE, EVENTS_CHECKIN_MANAGE, EVENTS_ANNOUNCEMENTS_MANAGE, EVENTS_INCIDENTS_MANAGE to the MEMBER role.
(MEMBER should not get COMPLIANCE_MANAGE or SURVEYS_MANAGE — those are admin-only.)

- EventCheckInSummary (eventProjectId, checkedIn count, total count)
- EventIncidentWithParticipants (incident + participant names)
- ParticipantDocumentStatus (registrationId, participantName, requirements array with isComplete per item)

These types are consumed by API routes (response shaping) and UI components (prop types).
