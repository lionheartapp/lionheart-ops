  <name>Task 1: Draft events [id] route + room conflict detection</name>
  <files>src/app/api/draft-events/[id]/route.ts, src/lib/services/eventService.ts, src/lib/services/draftEventService.ts</files>
  <action>
1. **Create `src/app/api/draft-events/[id]/route.ts`** with GET, PUT, and DELETE handlers following the exact project pattern (see inventory/[id]/route.ts). The [id] directory already exists but is empty.

   - `type RouteParams = { params: Promise<{ id: string }> }` (Next.js 15 async params)
   - GET: `const { id } = await params`, call `draftEventService.getDraftEventById(id, ctx.userId)`, return 404 if null
   - PUT: parse `req.json()`, call `draftEventService.updateDraftEvent(id, body, ctx.userId)`, handle ZodError -> 400, not found -> 404, access denied -> 403
   - DELETE: call `draftEventService.deleteDraftEvent(id, ctx.userId)`, return `ok({ deleted: true })`, handle not found -> 404, access denied -> 403
   - All three handlers: `getOrgIdFromRequest(req)` + `getUserContext(req)` + `runWithOrgContext(orgId, ...)`
   - Error handling: `isAuthError` -> 401, "Insufficient permissions" or "Access denied" -> 403, "not found" -> 404, ZodError -> 400, fallback -> 500

2. **Add `checkRoomConflict` to `src/lib/services/eventService.ts`** as an unexported (private) async function:
   - Parameters: `room: string, startsAt: Date, endsAt: Date, excludeId?: string`
   - If `!room || !room.trim()` return early (no room = no conflict)
   - Query: `prisma.event.findFirst({ where: { room: { equals: room.trim(), mode: 'insensitive' }, status: { not: 'CANCELLED' }, startsAt: { lt: endsAt }, endsAt: { gt: startsAt }, ...(excludeId ? { NOT: { id: excludeId } } : {}) } })`
   - If conflict found, throw `new Error(...)` with `(err as any).code = 'ROOM_CONFLICT'` containing the conflicting event's time range

3. **Wire `checkRoomConflict` into `createEvent`**: After validation, before `prisma.event.create`, call `await checkRoomConflict(validated.room, new Date(validated.startsAt), new Date(validated.endsAt))` only when `validated.room` is truthy.

4. **Wire `checkRoomConflict` into `updateEvent`**: After validation, when room or time fields changed, call `await checkRoomConflict(...)` with `excludeId = id` to skip self-comparison. Use the existing event's values for any fields not being updated.

5. **Wire `checkRoomConflict` into `submitDraftEvent` in `src/lib/services/draftEventService.ts`**: Before creating the confirmed event from draft, call the imported `checkRoomConflict` (export it from eventService for this purpose, or inline the conflict query). When `draft.room` is truthy, call `await checkRoomConflict(draft.room, draft.startsAt, draft.endsAt)`. This prevents publishing a draft that conflicts with existing events.

6. **Handle ROOM_CONFLICT in event API routes**: In `src/app/api/events/route.ts` POST handler (and any route that calls createEvent/updateEvent), add catch for `(error as any).code === 'ROOM_CONFLICT'` returning `fail('ROOM_CONFLICT', error.message)` with `{ status: 409 }`.
