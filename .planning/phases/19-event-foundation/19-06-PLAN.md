  <files>src/lib/services/eventDashboardService.ts, src/app/api/events/dashboard/route.ts, src/lib/hooks/useEventDashboard.ts</files>
  <action>
    1. Create `src/lib/services/eventDashboardService.ts`:

       a. **Define types:**
       ```typescript
       type ActionItemType = 'overdue_task' | 'upcoming_deadline' | 'pending_approval' | 'missing_field' | 'no_schedule' | 'no_tasks'
       type ResolveAction = { type: 'complete_task'; taskId: string; eventProjectId: string }
         | { type: 'approve_event'; eventProjectId: string }
         | { type: 'navigate'; url: string }
       interface RawActionItem {
         id: string
         type: ActionItemType
         title: string
         description: string
         eventProjectId: string
         eventProjectTitle: string
         dueDate?: Date
         resolveAction: ResolveAction
       }
       interface ScoredActionItem extends RawActionItem {
         urgencyScore: number  // 1-10
         aiReason: string
       }
       ```

       b. **`collectRawActionItems(orgId: string)`** — queries active EventProjects (status IN [CONFIRMED, IN_PROGRESS]) with tasks and schedule blocks. Generates action items:
          - Overdue tasks: tasks with dueDate < now and status != DONE. Action: complete_task.
          - Upcoming deadlines: tasks due within 3 days. Action: navigate to task.
          - Pending approvals: projects with status PENDING_APPROVAL. Action: approve_event.
          - Missing schedule: confirmed projects with 0 schedule blocks and startsAt within 14 days. Action: navigate to schedule tab.
          - No tasks: confirmed projects with 0 tasks. Action: navigate to tasks tab.
          - Missing fields: confirmed projects where description is null or locationText is null. Action: navigate to overview.

       c. **`scoreActionItems(items: RawActionItem[])`** — Calls Gemini to rank items:
          - If `GEMINI_API_KEY` is not set, return items sorted by dueDate (soonest first) with urgencyScore=5 and aiReason="AI scoring unavailable".
          - If items.length === 0, return empty array.
          - If items.length <= 3, skip AI call (not worth it) — sort by type priority (overdue > pending_approval > upcoming > missing).
          - Build prompt (from research Code Examples): explain context (school event coordinator), provide items as JSON, ask for urgency 1-10 ranking with reasons, request JSON array response.
          - Parse response. Use try/catch — if Gemini returns non-JSON, fall back to rule-based sorting.
          - Merge scores with original items, sort by urgencyScore descending.

       d. **`getAIPrioritizedActions(orgId: string)`** — Composes collectRawActionItems + scoreActionItems. Returns ScoredActionItem[].

    2. Create `src/app/api/events/dashboard/route.ts`:
       - **GET**: Permission EVENT_PROJECT_READ. Optional query param `?skipAI=true` to get raw items without Gemini scoring (for fast initial load).
         - If skipAI: call collectRawActionItems, sort by dueDate, return immediately.
         - If !skipAI: call getAIPrioritizedActions, return scored items.
       - **POST** (for resolve actions): Permission EVENT_PROJECT_UPDATE_ALL. Body: `{ resolveAction: ResolveAction }`.
         - If type='complete_task': call updateEventTask(taskId, { status: 'DONE' }, ctx.userId, eventProjectId).
         - If type='approve_event': call approveEventProject(eventProjectId, ctx.userId).
         - Return ok({ resolved: true }).
       - Follow standard route pattern.

    3. Create `src/lib/hooks/useEventDashboard.ts`:
       - `useEventDashboard()` — Two-phase fetch pattern:
         - First query: GET /api/events/dashboard?skipAI=true (fast, no AI). StaleTime: 0 (always fresh). Query key: `['event-dashboard', 'raw']`.
         - Second query: GET /api/events/dashboard (with AI scoring). StaleTime: 60000 (1 min cache). Query key: `['event-dashboard', 'scored']`. Enabled only after raw data loads.
         - Return: `{ rawItems, scoredItems, isLoadingRaw, isLoadingScored }`. UI shows rawItems first, then swaps to scoredItems when available.
       - `useResolveAction()` — POST mutation to /api/events/dashboard. Invalidates dashboard queries on success. Optimistic update: remove resolved item from list immediately.

    IMPORTANT:
    - Gemini call should be wrapped in try/catch. If it fails, log the error and return rule-based sorting. Never let AI failure break the dashboard.
    - The API key check (`process.env.GEMINI_API_KEY || process.env.NEXT_PUBLIC_GEMINI_API_KEY`) must handle missing key gracefully.
    - Action item IDs should be deterministic (e.g., `overdue_task_${taskId}`) so the UI can track items across re-fetches.
