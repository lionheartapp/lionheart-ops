Repeat repair detection service that identifies three asset alert conditions (repeat repairs, cost threshold, end of life), generates an AI replace-vs-repair recommendation via Anthropic Claude, sends email alerts to the Head of Maintenance, and displays badges on asset detail and ticket cards.

Purpose: Automatically surfaces problematic assets so the Head of Maintenance can make data-driven repair vs. replace decisions without manual auditing.
Output: Working cron-driven detection, AI recommendation stored on asset, three email alert types, badge UI on asset detail and ticket cards.

3. **Create `src/lib/services/repeatRepairService.ts`**:

   **`runRepeatRepairDetection(orgId: string): Promise<{ repeatRepair: number, costThreshold: number, endOfLife: number }>`**:

   Step A — Load all active assets with their ticket history:
   ```typescript
   const assets = await rawPrisma.maintenanceAsset.findMany({
     where: { organizationId: orgId, deletedAt: null, status: { not: 'DECOMMISSIONED' } },
     include: {
       tickets: {
         where: { deletedAt: null, status: 'DONE' },
         include: {
           costEntries: { select: { amount: true } },
           laborEntries: {
             select: {
               durationMinutes: true,
               technician: { select: { technicianProfile: { select: { loadedHourlyRate: true } } } },
             },
           },
         },
       },
     },
   })
   ```

   Step B — Find Head of Maintenance recipients:
   ```typescript
   const recipients = await rawPrisma.user.findMany({
     where: {
       organizationId: orgId,
       deletedAt: null,
       userRole: {
         permissions: {
           some: {
             permission: {
               resource: 'maintenance',
               action: 'analytics',
               scope: 'view',
             },
           },
         },
       },
     },
     select: { id: true, email: true, firstName: true, lastName: true },
   })
   if (recipients.length === 0) return { repeatRepair: 0, costThreshold: 0, endOfLife: 0 }
   ```

   Step C — Process each asset:
   ```
   const now = new Date()
   const twelveMonthsAgo = new Date(now); twelveMonthsAgo.setFullYear(twelveMonthsAgo.getFullYear() - 1)
   const thirtyDaysAgo = new Date(now); thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30)
   ```

   **REPAIR-01 — Repeat Repair Detection:**
   - Tickets DONE in last 12 months: `asset.tickets.filter(t => new Date(t.updatedAt) >= twelveMonthsAgo)`
   - If `repairsInYear.length >= 3`:
     - Skip if `asset.repeatAlertSentAt && asset.repeatAlertSentAt >= thirtyDaysAgo` (idempotency)
     - Send alert emails to all recipients (fire-and-forget)
     - Create in-app notifications for all recipients
     - Update: `rawPrisma.maintenanceAsset.update({ where: { id: asset.id }, data: { repeatAlertSentAt: now } })`
     - Increment counter

   **REPAIR-02 — Cost Threshold Detection:**
   - Compute `cumulativeRepairCost` for the asset (same pattern as in `maintenanceAssetService.ts` `getAssetWithDetails`)
   - If `asset.replacementCost && cumulativeRepairCost >= asset.repairThresholdPct * asset.replacementCost`:
     - Skip if `asset.costAlertSentAt && asset.costAlertSentAt >= thirtyDaysAgo` (idempotency)
     - Check if AI recommendation already fresh (generated in last 30 days): `asset.aiRecommendationAt && asset.aiRecommendationAt >= thirtyDaysAgo`
     - If not fresh: call `generateReplaceVsRepairRecommendation(asset, cumulativeRepairCost)`, store result
     - Send cost threshold alert emails including recommendation text
     - Create in-app notifications
     - Update: `rawPrisma.maintenanceAsset.update({ where: { id: asset.id }, data: { costAlertSentAt: now, aiRecommendation: recommendation, aiRecommendationAt: now } })`
     - Increment counter

   **REPAIR-03 — End of Life Detection:**
   - If `asset.expectedLifespanYears && asset.purchaseDate`:
     - `const ageYears = (now.getTime() - new Date(asset.purchaseDate).getTime()) / (365.25 * 24 * 3600 * 1000)`
     - If `ageYears >= asset.expectedLifespanYears`:
       - Skip if `asset.eolAlertSentAt && asset.eolAlertSentAt >= thirtyDaysAgo` (idempotency)
       - Send end of life alert emails
       - Create in-app notifications
       - Update: `rawPrisma.maintenanceAsset.update({ where: { id: asset.id }, data: { eolAlertSentAt: now } })`
       - Increment counter

   **`generateReplaceVsRepairRecommendation(asset, cumulativeRepairCost): Promise<object>`**:
   - Requires `process.env.ANTHROPIC_API_KEY` — if missing, return `{ recommendation: 'AI recommendation unavailable — ANTHROPIC_API_KEY not configured', generatedAt: new Date().toISOString() }`
   - Build prompt:
     ```
     You are a facilities management expert. Analyze this asset and provide a replace-vs-repair recommendation.

     Asset: [asset.name] ([asset.make] [asset.model])
     Category: [asset.category]
     Purchase Date: [asset.purchaseDate]
     Expected Lifespan: [asset.expectedLifespanYears] years
     Replacement Cost: $[asset.replacementCost]
     Cumulative Repair Cost: $[cumulativeRepairCost] ([pct]% of replacement cost)
     Repair Threshold: [asset.repairThresholdPct * 100]%

     Provide a concise recommendation (2-3 sentences) covering:
     1. Whether to repair or replace
     2. Key financial reasoning
     3. Suggested timeline for action

     Respond with JSON: { "recommendation": "...", "decision": "REPLACE" | "REPAIR", "urgency": "LOW" | "MEDIUM" | "HIGH" }
     ```
   - Call Anthropic API:
     ```typescript
     import Anthropic from '@anthropic-ai/sdk'
     const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })
     const response = await anthropic.messages.create({
       model: 'claude-sonnet-4-5',
       max_tokens: 512,
       messages: [{ role: 'user', content: prompt }],
     })
     const text = response.content[0].type === 'text' ? response.content[0].text : ''
     return JSON.parse(text)
     ```
   - Wrap in try/catch — if JSON parse fails or API errors, return `{ recommendation: text || 'Failed to generate recommendation', decision: 'UNKNOWN', urgency: 'MEDIUM' }`

4. **Extend cron handler** (`src/app/api/cron/maintenance-tasks/route.ts`):

   ```typescript
   import { runRepeatRepairDetection } from '@/lib/services/repeatRepairService'

   let repairDetectedCount = 0

   // ── Task 3: Repeat repair detection ────────────────────────────────
   try {
     // Get distinct org IDs that have the maintenance module enabled
     const orgsWithMaintenance = await rawPrisma.tenantModule.findMany({
       where: { moduleKey: 'maintenance', isEnabled: true },
       select: { organizationId: true },
     })
     for (const { organizationId } of orgsWithMaintenance) {
       try {
         const result = await runRepeatRepairDetection(organizationId)
         repairDetectedCount += result.repeatRepair + result.costThreshold + result.endOfLife
       } catch (orgErr) {
         console.error(`[cron] Repair detection failed for org ${organizationId}:`, orgErr)
       }
     }
     console.log(`[cron/maintenance-tasks] Repair detections triggered: ${repairDetectedCount}`)
   } catch (repairErr) {
     console.error('[cron/maintenance-tasks] Repair detection task failed:', repairErr)
   }
   ```
