<objective>
Close three auth security gaps identified in the v2.0 milestone audit:
1. Rate-limit the reset-password endpoint (currently missing from middleware rate limiting)
2. Set httpOnly auth + CSRF cookies on signup response (matching login pattern)
3. Remove localStorage JWT writes from signup page and SignupModal

Purpose: Ensure uniform security posture across all auth endpoints — no public endpoint bypasses rate limiting, and no auth flow falls back to localStorage for JWT storage.
Output: Patched middleware, signup API route, signup page, and SignupModal.

**1. Update signup page (src/app/signup/page.tsx):**

a. Add `credentials: 'include'` to the fetch call so the browser accepts the Set-Cookie header from the signup response:

```typescript
const res = await fetch('/api/organizations/signup', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'include',
  body: JSON.stringify({ ... }),
})
```

b. In the success handler (the `if (data.data)` block), REMOVE all `localStorage.setItem` calls for `auth-token`, `org-id`, `org-slug`. Keep `org-name`, `user-name`, `user-email` in localStorage since they are non-sensitive display data used by onboarding pages and the AuthBridge populates them from /api/auth/me anyway.

Replace the block with:
```typescript
if (data.data) {
  const { slug } = data.data
  // Non-sensitive display data for onboarding pages (AuthBridge will refresh from /api/auth/me)
  localStorage.setItem('org-name', schoolName)
  localStorage.setItem('org-slug', slug || '')
  localStorage.setItem('user-name', name)
  localStorage.setItem('user-email', email)

  // Auth token is now in httpOnly cookie set by the server — no localStorage write needed
  // Redirect to onboarding
  window.location.href = '/onboarding/school-info'
}
```

**2. Update SignupModal (src/app/SignupModal.tsx):**

Apply the same two changes:

a. Add `credentials: 'include'` to the fetch call.

b. Remove `localStorage.setItem('auth-token', ...)` and `localStorage.setItem('org-id', ...)`. Keep `org-name`, `org-slug`, `user-name`, `user-email` for display.

Replace the success block with:
```typescript
if (data.data) {
  const { slug } = data.data
  localStorage.setItem('org-name', schoolName)
  localStorage.setItem('org-slug', slug || '')
  localStorage.setItem('user-name', name)
  localStorage.setItem('user-email', email)

  // Auth token is now in httpOnly cookie — no localStorage write needed
  window.location.href = '/onboarding/school-info'
}
```

**Why we keep some localStorage writes:** The onboarding pages (`/onboarding/school-info`, `/onboarding/members`, `/onboarding/setup`) read `org-name`, `user-name`, etc. from localStorage for display. The AuthBridge in providers.tsx will populate these from /api/auth/me on subsequent page loads. Removing these display-only values would break the immediate redirect to onboarding. The critical change is that `auth-token` (the JWT) is NO LONGER stored in localStorage — it lives only in the httpOnly cookie.
