**Extend src/lib/services/ai/eventAIService.ts with new functions:**

- `generateRegistrationForm(params: { eventType: string, durationDays: number, expectedAttendance: number, description?: string })` — Returns { sections: { title: string, fields: { type: string, label: string, helpText?: string, required: boolean, options?: string[] }[] }[] }. Prompts Gemini to suggest appropriate form fields for the event type (e.g., camp needs medical + dietary + t-shirt, field trip needs permission + emergency contact only). Returns null if no API key.

- `generateGroupAssignments(params: { participants: { id: string, name: string, grade?: string, gender?: string, friendRequests?: string[] }[], groups: { id: string, name: string, type: string, capacity: number }[], constraints: { balanceGender?: boolean, balanceGrade?: boolean, honorFriendRequests?: boolean, counselorRatio?: number } })` — Returns { assignments: { participantId: string, groupId: string, reasoning?: string }[], warnings: string[] }. AI considers constraints and generates balanced assignments. Returns null if no API key.

- `detectConflicts(params: { eventProjectId: string, startsAt: string, endsAt: string, buildingId?: string, roomId?: string, organizationId: string })` — Queries existing EventProjects and CalendarEvents in the date range. Checks for: room double-booking, staff scheduling conflicts (same createdBy has overlapping event), transportation overlap (events at external venues on same day), audience overlap (same school/grade has two events). Returns { conflicts: { type: string, severity: 'high' | 'medium' | 'low', description: string, conflictingEventId?: string, conflictingEventTitle?: string }[] }. This function does NOT need AI — it's a deterministic query + logic. Uses rawPrisma since it needs cross-event access.

- `estimateBudgetFromHistory(params: { eventType?: string, durationDays: number, expectedAttendance: number, organizationId: string })` — Queries historical BudgetLineItem data for similar events in the org. If enough data (3+ past events), calculates average per-category amounts adjusted for attendance. If insufficient data, falls back to Gemini estimation. Returns AIBudgetEstimate.

- `analyzeFeedback(surveyResponses: { question: string, answers: string[] }[])` — Already defined in Plan 03. Ensure it's fully implemented: sends survey data to Gemini, requests theme extraction with sentiment analysis. Returns { themes, summary, actionItems }.

**Create API routes (all follow standard pattern):**

1. **src/app/api/events/ai/generate-form/route.ts**
   - POST: assertCan(EVENTS_REGISTRATION_MANAGE). Body: { eventType, durationDays, expectedAttendance, description? }. Returns generated form structure.

2. **src/app/api/events/ai/generate-groups/route.ts**
   - POST: assertCan(EVENTS_GROUPS_MANAGE). Body: { eventProjectId }. Service loads participants and groups from DB, then calls generateGroupAssignments. Returns assignments + warnings.

3. **src/app/api/events/ai/detect-conflicts/route.ts**
   - POST: assertCan(EVENTS_CREATE). Body: { eventProjectId } or { startsAt, endsAt, buildingId?, roomId? }. If eventProjectId provided, loads event data. Calls detectConflicts. Returns conflicts array.

4. **src/app/api/events/ai/estimate-budget/route.ts**
   - POST: assertCan(EVENTS_BUDGET_READ). Body: { eventProjectId } or { eventType, durationDays, expectedAttendance }. If eventProjectId, loads event data. Calls estimateBudgetFromHistory. Returns budget estimate.

5. **src/app/api/events/ai/analyze-feedback/route.ts**
   - POST: assertCan(EVENTS_SURVEYS_MANAGE). Body: { eventProjectId }. Loads surveys and responses for the event. Calls analyzeFeedback. Returns themes + summary + action items.
