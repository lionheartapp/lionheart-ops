From src/lib/logger.ts:
```typescript
export const logger = process.env.NODE_ENV === 'production'
  ? pino({ level: 'info' })
  : pino({ transport: { target: 'pino-pretty', options: { colorize: true } }, level: 'debug' })
```

Pattern to apply to every handler function:
```typescript
import { logger } from '@/lib/logger'
import * as Sentry from '@sentry/nextjs'

export async function GET(req: NextRequest) {
  const log = logger.child({ route: '/api/example', method: 'GET' })
  try {
    const orgId = getOrgIdFromRequest(req)
    Sentry.setTag('org_id', orgId)
    // ... existing business logic unchanged ...
  } catch (error) {
    log.error({ err: error }, 'Failed to [action description]')
    Sentry.captureException(error)
    return NextResponse.json(fail('INTERNAL_ERROR', '...'), { status: 500 })
  }
}
```

CRITICAL RULES:
- `const log = logger.child(...)` BEFORE the try block (so catch can access it)
- `Sentry.setTag('org_id', orgId)` AFTER orgId extraction (only if orgId exists in that handler)
- `log.error({ err: error }, 'message')` — use `err` key for Pino error serializer
- `Sentry.captureException(error)` — in every catch block
- NEVER log request bodies, emails, names (FERPA compliance)
- NEVER import logger in middleware.ts (Edge runtime incompatible)
- Public routes (public/contact) and auth routes (auth/me, auth/logout, auth/resend-verification) may lack orgId — skip Sentry.setTag('org_id') if orgId is not available
- Billing routes use rawPrisma — do NOT change their Prisma imports
