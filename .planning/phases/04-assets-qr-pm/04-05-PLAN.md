1. **Labor/cost service** (`src/lib/services/laborCostService.ts`):
   - `createLaborEntry(orgId, ticketId, data)`: Zod validation. Fields: technicianId (required — defaults to current user), startTime (DateTime, required), endTime (DateTime, optional), durationMinutes (Int, optional — computed from start/end if both present, or set manually), notes (optional). Auto-compute: if startTime and endTime provided, compute durationMinutes = diff in minutes. Look up TechnicianProfile.loadedHourlyRate for the technician to compute labor cost on read.
   - `getLaborEntries(ticketId)`: List all for ticket, include technician name and loadedHourlyRate. Order by startTime desc.
   - `updateLaborEntry(orgId, entryId, data)`: Partial update (edit end time, notes).
   - `deleteLaborEntry(orgId, entryId)`: Hard delete (labor entries are not soft-deleted).
   - `createCostEntry(orgId, ticketId, data)`: Fields: vendor (String, required), description (String, required), amount (Float, required), receiptUrl (String, optional). createdById = current user.
   - `getCostEntries(ticketId)`: List all for ticket, include createdBy name. Order by createdAt desc.
   - `updateCostEntry(orgId, entryId, data)`: Partial update.
   - `deleteCostEntry(orgId, entryId)`: Hard delete.
   - `getCostSummary(ticketId)`: Aggregate: totalLaborHours (sum of durationMinutes/60), laborCost (sum of hours * each tech's loadedHourlyRate), materialsCost (sum of costEntry amounts), grandTotal (laborCost + materialsCost). Returns object with all 4 values.
   - `getVendorList(orgId)`: SELECT DISTINCT vendor FROM MaintenanceCostEntry WHERE organizationId = orgId ORDER BY vendor. Returns string[].

2. **Labor API routes**:
   - `GET /api/maintenance/tickets/[id]/labor` — List entries. Permission: MAINTENANCE_CLAIM
   - `POST /api/maintenance/tickets/[id]/labor` — Create entry. Permission: MAINTENANCE_CLAIM
   - `PATCH /api/maintenance/tickets/[id]/labor/[entryId]` — Update. Permission: MAINTENANCE_CLAIM
   - `DELETE /api/maintenance/tickets/[id]/labor/[entryId]` — Delete. Permission: MAINTENANCE_CLAIM

3. **Cost API routes**:
   - `GET /api/maintenance/tickets/[id]/costs` — List entries. Permission: MAINTENANCE_CLAIM
   - `POST /api/maintenance/tickets/[id]/costs` — Create entry. Permission: MAINTENANCE_CLAIM
   - `PATCH /api/maintenance/tickets/[id]/costs/[entryId]` — Update. Permission: MAINTENANCE_CLAIM
   - `DELETE /api/maintenance/tickets/[id]/costs/[entryId]` — Delete. Permission: MAINTENANCE_CLAIM

4. **Receipt upload** (`POST /api/maintenance/tickets/[id]/cost-upload-url`):
   - Signed URL for Supabase Storage bucket `maintenance-receipts`
   - Path: `{orgId}/{ticketId}/{timestamp}-{filename}`
   - Same pattern as existing maintenance-photos upload-url route

5. **Vendor autocomplete** (`GET /api/maintenance/vendors`):
   - Returns distinct vendor strings from org's cost entries
   - Optional `?q=` param for prefix filtering (ILIKE)
   - Permission: MAINTENANCE_CLAIM

6. **Add estimatedRepairCostUSD** — Verify the field added in Plan 01 schema push exists on MaintenanceTicket. If the existing ticket PATCH route doesn't handle this field, add it to the allowed update fields.
