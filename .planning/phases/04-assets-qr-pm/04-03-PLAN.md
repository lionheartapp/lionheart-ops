1. **PM schedule service** (`src/lib/services/pmScheduleService.ts`):
   - `computeNextDueDate(baseDate, recurrenceType, intervalDays?, months?)`: Using date-fns — DAILY: addDays(1), WEEKLY: addWeeks(1), BIWEEKLY: addWeeks(2), MONTHLY: addMonths(1), QUARTERLY: addMonths(3), SEMIANNUAL: addMonths(6), ANNUAL: addYears(1), CUSTOM: addDays(intervalDays ?? 30). For MONTHLY with specific months array, find next month in the array after baseDate's month.
   - `createPmSchedule(orgId, data)`: Zod validation. Fields: name (required), description, recurrenceType (required, one of DAILY/WEEKLY/BIWEEKLY/MONTHLY/QUARTERLY/SEMIANNUAL/ANNUAL/CUSTOM), intervalDays (required if CUSTOM), months (Int[] for specific month selection), advanceNoticeDays (default 7), checklistItems (String[]), assetId (optional), buildingId/areaId/roomId (optional location), schoolId (optional), defaultTechnicianId (optional), avoidSchoolYear (boolean, default false). Compute initial nextDueDate from today. Create via prisma.
   - `getPmSchedules(orgId, filters)`: List with filters: assetId, buildingId, status (active/paused), keyword. Include asset name, location names, technician name.
   - `getPmScheduleById(orgId, id)`: Full detail with all relations.
   - `updatePmSchedule(orgId, id, data)`: Partial update. If recurrence fields change, recompute nextDueDate.
   - `deletePmSchedule(orgId, id)`: Soft-delete.
   - PM schedule status values (constants): ACTIVE, PAUSED.

   - `GET /api/maintenance/pm-schedules/[id]` — Detail. Permission: ASSETS_READ
   - `PATCH /api/maintenance/pm-schedules/[id]` — Update. Permission: ASSETS_UPDATE
   - `DELETE /api/maintenance/pm-schedules/[id]` — Delete. Permission: ASSETS_DELETE
   - All follow standard route pattern.
