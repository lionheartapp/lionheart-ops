1. **PM ticket generation function** (add to `pmScheduleService.ts`):
   - `generatePmTickets()`: Called by cron. Uses `rawPrisma` (no org context — iterates all orgs like existing cron pattern).
   - Query all active PmSchedules where `nextDueDate <= today + advanceNoticeDays`
   - For each qualifying schedule:
     - Create MaintenanceTicket with: status='TODO', title=schedule.name, description=schedule.description, category from asset or 'OTHER', pmScheduleId=schedule.id, pmScheduledDueDate=schedule.nextDueDate, pmChecklistItems=schedule.checklistItems (copy array), pmChecklistDone=schedule.checklistItems.map(() => false), assignedToId=schedule.defaultTechnicianId, assetId=schedule.assetId, buildingId/areaId/roomId from schedule, organizationId=schedule.organizationId
     - Generate ticket number using existing MaintenanceCounter pattern
     - Wrap in try/catch: if UniqueConstraintError on (pmScheduleId, pmScheduledDueDate), skip silently (idempotent)
   - Return count of tickets created for logging

2. **Extend cron route** (`src/app/api/cron/maintenance-tasks/route.ts`):
   - Import and call `generatePmTickets()` alongside existing cron tasks
   - Log: "PM tickets generated: {count}"
   - Ensure cron is idempotent end-to-end

3. **QA transition gate for PM tickets** (modify `maintenanceTicketService.ts`):
   - In the `performStatusTransition` function (or equivalent), when transitioning to QA status:
   - If `ticket.pmScheduleId != null && ticket.pmChecklistItems.length > 0`:
     - Check `ticket.pmChecklistDone.every(Boolean)`
     - If not all done, throw Error: "Complete all PM checklist items before moving to QA"
   - This is a SERVER-SIDE gate — cannot be bypassed by direct API call

4. **Next-due-date recalculation on completion** (modify `maintenanceTicketService.ts`):
   - When a PM ticket transitions to DONE status:
   - Fetch the linked PmSchedule via ticket.pmScheduleId
   - Call `computeNextDueDate(new Date(), schedule.recurrenceType, schedule.intervalDays, schedule.months)` — use completion date (today), NOT the scheduled date
   - Update PmSchedule.nextDueDate and PmSchedule.lastCompletedDate = today
   - This ensures PM cycles forward from when work was actually completed

5. **Checklist toggle API** (`src/app/api/maintenance/tickets/[id]/checklist/route.ts`):
   - `PATCH` — Body: `{ index: number, done: boolean }`. Updates pmChecklistDone[index] to the given value.
   - Permission: MAINTENANCE_CLAIM (technician or head can toggle)
   - Validate index is within bounds of pmChecklistItems array
   - Return updated pmChecklistItems and pmChecklistDone arrays
