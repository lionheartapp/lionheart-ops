<!-- API contracts the UI consumes -->

GET /api/inventory?search=term&category=Office+Supplies
Response: { ok: true, data: InventoryItem[] }

POST /api/inventory
Body: { name, category?, sku?, quantityOnHand?, reorderThreshold? }
Response: { ok: true, data: InventoryItem }

GET /api/inventory/[id]
Response: { ok: true, data: InventoryItem }

PUT /api/inventory/[id]
Body: Partial<{ name, category, sku, quantityOnHand, reorderThreshold }>
Response: { ok: true, data: InventoryItem }

DELETE /api/inventory/[id]
Response: { ok: true, data: { deleted: true } }

POST /api/inventory/[id]/checkout
Body: { quantity: number, dueDate?: string, notes?: string }
Response: { ok: true, data: InventoryItem } | { ok: false, error: { code: 'INSUFFICIENT_STOCK' } }

POST /api/inventory/[id]/checkin
Body: { transactionId: string, notes?: string }
Response: { ok: true, data: InventoryItem }

GET /api/inventory/[id]/transactions
Response: { ok: true, data: InventoryTransaction[] }

InventoryItem shape:
{ id, name, category, sku, quantityOnHand, reorderThreshold, createdAt, updatedAt }

InventoryTransaction shape:
{ id, itemId, type, quantity, checkedOutBy?: {id, firstName, lastName, email}, checkedInBy?: {id, firstName, lastName, email}, dueDate?, checkedOutAt?, checkedInAt?, notes?, createdAt }
</interfaces>

<task type="auto">
  <name>Task 1: Build complete inventory management page</name>
  <files>src/app/inventory/page.tsx</files>
  <action>
Create `src/app/inventory/page.tsx` as a `'use client'` component. Follow established UI patterns from the codebase (glassmorphism, animations, TanStack Query).

**Page structure:**

1. **Header section:**
   - Page title "Inventory" with stagger animation (use `fadeInUp` from `@/lib/animations`)
   - Summary stat cards in a row: Total Items, Total In Stock, Low Stock (items below threshold), Checked Out (items with open checkouts). Use `ui-glass` card style with `AnimatedCounter` for numbers. Low Stock card should use warning styling (amber/orange accent) when count > 0.
   - "Add Item" button (primary dark pill style from MEMORY.md: `px-5 py-2.5 rounded-full bg-gray-900 text-white...`)

2. **Search and filter bar:**
   - Search input with `onChange` debounced (300ms) — searches by name
   - Category dropdown filter — options from `INVENTORY_CATEGORIES` constant plus "All Categories" default. Use a styled select or the `FloatingDropdown` pattern if available.
   - Stock level filter: "All", "Low Stock", "In Stock", "Out of Stock"

3. **Items table:**
   - Container: `ui-glass-table`
   - Columns: Name, Category, SKU, Quantity, Reorder Point, Status, Actions
   - **Status column**: Show a colored badge:
     - Green "In Stock" when `quantityOnHand > reorderThreshold`
     - Amber "Low Stock" when `quantityOnHand > 0 && quantityOnHand <= reorderThreshold`
     - Red "Out of Stock" when `quantityOnHand === 0`
   - **Actions column**: "Checkout" button (enabled when quantity > 0), "View" button to open detail drawer
   - Row click opens the detail drawer for that item
   - Hover state on rows: `hover:bg-gray-50/50 transition-colors duration-200`
   - Empty state: If no items, show centered empty state with Package icon + "No inventory items yet" + "Add your first item to start tracking stock" + "Add Item" CTA button

4. **Skeleton loading:**
   - Show `animate-pulse` skeleton matching the table layout during initial fetch
   - 5 skeleton rows with rounded blocks matching column widths

5. **Add/Edit Item Drawer (or Modal):**
   - Slide-in drawer from the right (use existing `DetailDrawer` component or build inline with AnimatePresence)
   - Form fields: Name (required), Category (dropdown from INVENTORY_CATEGORIES), SKU (optional text), Quantity on Hand (number input, min 0), Reorder Threshold (number input, min 0)
   - Save button calls POST /api/inventory (create) or PUT /api/inventory/[id] (edit)
   - On success: invalidate query cache, close drawer, show success feedback
   - Validation errors from API shown inline on form fields

6. **Item Detail Drawer:**
   - Opens when clicking "View" or clicking a row
   - Shows item details at top: name, category, sku, quantity (large number), reorder threshold
   - **Quick actions section:** "Checkout" and "Edit" and "Delete" buttons
   - **Checkout form (inline in drawer):** Quantity (number, min 1, max item.quantityOnHand), Due Date (optional date picker or date input), Notes (optional textarea). Submit calls POST /api/inventory/[id]/checkout. On INSUFFICIENT_STOCK error, show inline error message.
   - **Transaction history section:** Fetches GET /api/inventory/[id]/transactions. Shows a timeline/list of transactions:
     - CHECKOUT entries: red-ish indicator, "{firstName lastName} checked out {qty}" with date. If `checkedInAt` is null, show "Outstanding" badge. Show "Check In" button for open checkouts.
     - CHECKIN entries: green-ish indicator, "{firstName lastName} checked in {qty}" with date.
     - ADJUSTMENT entries: blue indicator, "Adjustment: {qty}" with date.
   - **Check In action:** Clicking "Check In" on an open checkout calls POST /api/inventory/[id]/checkin with that transaction's id. On success, invalidate queries.

7. **Delete confirmation:**
   - Use `ConfirmDialog` component. "Are you sure you want to delete {itemName}? This action cannot be undone."
   - On confirm: call DELETE /api/inventory/[id], invalidate cache, close drawer.

**Data fetching with TanStack Query:**
- `useQuery(['inventory', { search, category, stockFilter }], ...)` for item list — use `fetchApi` from `@/lib/api-client` for authenticated requests
- `useQuery(['inventory', itemId], ...)` for single item detail (when drawer opens)
- `useQuery(['inventory-transactions', itemId], ...)` for transaction log (when drawer opens)
- `useMutation` for create, update, delete, checkout, checkin — each invalidates relevant queries on success

**Client-side stock level filtering:**
- "Low Stock" filter: `items.filter(i => i.quantityOnHand > 0 && i.quantityOnHand <= i.reorderThreshold)`
- "Out of Stock" filter: `items.filter(i => i.quantityOnHand === 0)`
- "In Stock" filter: `items.filter(i => i.quantityOnHand > i.reorderThreshold)`

**Styling requirements:**
- Use `ui-glass`, `ui-glass-table`, `ui-glass-hover` utility classes
- Card style: `bg-white border border-gray-200 rounded-xl p-6` or glassmorphism equivalent
- All clickable elements get `cursor-pointer`
- Transitions: `transition-colors duration-200` on interactive elements
- Mobile responsive: table scrolls horizontally on small screens, or switches to card layout on mobile
- Framer Motion: stagger entrance on stat cards and table rows using variants from `@/lib/animations`
