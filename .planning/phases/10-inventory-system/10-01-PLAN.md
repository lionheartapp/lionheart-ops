1. In `prisma/schema.prisma`, extend the existing `InventoryItem` model (line 549):
   - Add `category String?` field (e.g. "Office Supplies", "AV Equipment", "Custodial")
   - Add `sku String?` field (stock-keeping unit — optional free text)
   - Add `transactions InventoryTransaction[]` relation
   - Add index `@@index([organizationId, category])` for category filter queries
   - Keep existing fields and constraints unchanged

2. Add new `InventoryTransactionType` enum:
   ```
   enum InventoryTransactionType {
     CHECKOUT
     CHECKIN
     ADJUSTMENT
   }
   ```

3. Add new `InventoryTransaction` model:
   ```
   model InventoryTransaction {
     id               String                    @id @default(cuid())
     organizationId   String
     itemId           String
     type             InventoryTransactionType
     quantity         Int                       // negative for CHECKOUT, positive for CHECKIN, signed for ADJUSTMENT
     checkedOutById   String?
     checkedInById    String?
     dueDate          DateTime?
     checkedOutAt     DateTime?
     checkedInAt      DateTime?
     notes            String?
     createdAt        DateTime                  @default(now())

     organization Organization  @relation(fields: [organizationId], references: [id], onDelete: Cascade)
     item         InventoryItem @relation(fields: [itemId], references: [id], onDelete: Cascade)
     checkedOutBy User?         @relation("InventoryCheckedOutBy", fields: [checkedOutById], references: [id], onDelete: SetNull)
     checkedInBy  User?         @relation("InventoryCheckedInBy", fields: [checkedInById], references: [id], onDelete: SetNull)

     @@index([organizationId, itemId])
     @@index([organizationId, itemId, createdAt(sort: Desc)])
     @@index([itemId, checkedInAt])
   }
   ```

4. Add `InventoryTransaction[]` relation to `Organization` model (`inventoryTransactions InventoryTransaction[]`)

5. Add two named relations to `User` model:
   - `inventoryCheckouts  InventoryTransaction[] @relation("InventoryCheckedOutBy")`
   - `inventoryCheckins   InventoryTransaction[] @relation("InventoryCheckedInBy")`

6. In `src/lib/db/index.ts`, add `'InventoryTransaction'` to the `orgScopedModels` Set (under the `// Inventory` comment or near `InventoryItem`). Do NOT add it to `softDeleteModels` — transaction rows are immutable audit records.

7. Run `npm run db:push` to push schema changes to local DB.

3. Create `src/lib/services/inventoryService.ts` with the following structure:

   **Imports:** `z` from zod, `prisma`/`rawPrisma` from `@/lib/db`, `runWithOrgContext` from `@/lib/org-context`, `createBulkNotifications` from `notificationService`, `PERMISSIONS`/`matchesPermission` from `@/lib/permissions`, `stripAllHtml` from `@/lib/sanitize`

   **Category constants:**
   ```typescript
   export const INVENTORY_CATEGORIES = [
     'Office Supplies',
     'AV Equipment',
     'Custodial',
     'Sports Equipment',
     'Classroom Materials',
     'Technology',
     'Medical/First Aid',
     'Other',
   ] as const
   ```

   **Zod schemas:**
   - `CreateItemSchema`: name (string, 1-200 chars, transform with stripAllHtml), category (string, optional), sku (string, optional, max 100), quantityOnHand (int, min 0, default 0), reorderThreshold (int, min 0, default 0)
   - `UpdateItemSchema`: CreateItemSchema.partial()
   - `CheckoutSchema`: quantity (int, min 1), dueDate (string datetime, optional), notes (string, optional, transform with stripAllHtml)
   - `CheckinSchema`: transactionId (string, required), notes (string, optional, transform with stripAllHtml)

   **CRUD functions (all operate inside runWithOrgContext):**
   - `listItems(orgId, filters?)` — accepts optional `search` (name contains, insensitive) and `category` filter. Returns items ordered by name asc.
   - `getItem(orgId, itemId)` — findFirst by id, throws NOT_FOUND if missing
   - `createItem(orgId, input)` — validates with CreateItemSchema, creates via prisma.inventoryItem.create
   - `updateItem(orgId, itemId, input)` — validates with UpdateItemSchema, verifies item exists first, updates via prisma.inventoryItem.update
   - `deleteItem(orgId, itemId)` — verifies item exists, deletes via prisma.inventoryItem.delete (soft-delete via extension)

   **Checkout function:**
   - `checkoutItem(orgId, itemId, input, actorUserId)` — validates with CheckoutSchema. Inside runWithOrgContext:
     1. Fetch item, throw NOT_FOUND if missing
     2. Check `item.quantityOnHand >= input.quantity`, throw INSUFFICIENT_STOCK if not
     3. Use Prisma atomic `decrement`: `prisma.inventoryItem.update({ where: { id: itemId }, data: { quantityOnHand: { decrement: input.quantity } } })`
     4. Guard: if `updated.quantityOnHand < 0`, increment back and throw INSUFFICIENT_STOCK (race condition safety)
     5. Create InventoryTransaction row with type CHECKOUT, quantity as negative, checkedOutById, checkedOutAt: new Date(), dueDate, notes
     6. Fire-and-forget: if `updated.quantityOnHand < updated.reorderThreshold`, call `void notifyLowStock(orgId, item.name, updated.quantityOnHand, itemId)`
     7. Return the updated item

   **Checkin function:**
   - `checkinItem(orgId, input, actorUserId)` — validates with CheckinSchema. Inside runWithOrgContext:
     1. Find the transaction by transactionId, verify it exists, is type CHECKOUT, has `checkedInAt === null` (still open), and belongs to an item in this org
     2. Throw NOT_FOUND if transaction doesn't exist, ALREADY_CHECKED_IN if checkedInAt is set
     3. Get the absolute quantity from the transaction (Math.abs(transaction.quantity))
     4. Increment item quantity: `prisma.inventoryItem.update({ where: { id: transaction.itemId }, data: { quantityOnHand: { increment: Math.abs(transaction.quantity) } } })`
     5. Stamp the transaction: `prisma.inventoryTransaction.update({ where: { id: input.transactionId }, data: { checkedInAt: new Date(), checkedInById: actorUserId, notes: input.notes ?? transaction.notes } })`
     6. Return the updated item

   **Transaction log function:**
   - `getTransactions(orgId, itemId)` — Inside runWithOrgContext, findMany with `where: { itemId }`, `orderBy: { createdAt: 'desc' }`, include checkedOutBy and checkedInBy (select id, firstName, lastName, email)

   **Low-stock notification helper (private, NOT exported):**
   - `notifyLowStock(orgId, itemName, currentQty, itemId)` — wraps in try/catch (never throws):
     1. Find users with INVENTORY_UPDATE permission using the same `getUsersWithPermission` pattern from maintenanceNotificationService (use rawPrisma to find users whose role has inventory:update or *:* permission)
     2. Call `createBulkNotifications` with type `'inventory_low_stock'`, title `Low stock: ${itemName}`, body `Only ${currentQty} unit(s) remaining — below reorder threshold`, linkUrl `/inventory?highlight=${itemId}`
     3. Catch errors and console.error — never throw

   **getUsersWithPermission helper (private):** Copy the pattern from maintenanceNotificationService.ts — use rawPrisma to query users with a specific permission in an org. Include the `permissionToWhere` helper logic (parse permission string into resource/action/scope parts for the Prisma where clause). Import `matchesPermission` is not needed here — use the DB query approach.
