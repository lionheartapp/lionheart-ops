**Replace** the existing stub `src/app/api/inventory/route.ts` entirely. Every route handler follows the canonical CLAUDE.md boilerplate:
1. `getOrgIdFromRequest(req)` to read x-org-id header
2. `getUserContext(req)` to verify JWT
3. `assertCan(ctx.userId, PERMISSIONS.XXX)` for permission check
4. `runWithOrgContext(orgId, async () => { ... })` wrapping all DB work
5. `ok()` / `fail()` response envelope
6. Catch block: check for 'Insufficient permissions' message -> 403, otherwise 500

**Route 1: `src/app/api/inventory/route.ts`**
- `GET`: Permission `INVENTORY_READ`. Read `search` and `category` from URL searchParams. Call `listItems(orgId, { search, category })`. Return `ok(items)`.
- `POST`: Permission `INVENTORY_CREATE`. Parse JSON body, validate with `CreateItemSchema.safeParse`. On validation failure, return `fail('VALIDATION_ERROR', ...)` with status 400 and ZodError issues as details. On success, call `createItem(orgId, validated.data)`. Return `ok(item)` with status 201.

**Route 2: `src/app/api/inventory/[id]/route.ts`**
- Use `type RouteParams = { params: Promise<{ id: string }> }` (Next.js 15 pattern — params is a Promise)
- `GET`: Permission `INVENTORY_READ`. Call `getItem(orgId, id)`. Catch NOT_FOUND -> return `fail('NOT_FOUND', ...)` with status 404.
- `PUT`: Permission `INVENTORY_UPDATE`. Parse body, validate with `UpdateItemSchema.safeParse`. Call `updateItem(orgId, id, validated.data)`. Catch NOT_FOUND -> 404.
- `DELETE`: Permission `INVENTORY_DELETE`. Call `deleteItem(orgId, id)`. Catch NOT_FOUND -> 404. Return `ok({ deleted: true })`.

**Route 3: `src/app/api/inventory/[id]/transactions/route.ts`**
- `GET`: Permission `INVENTORY_READ`. Call `getTransactions(orgId, id)`. Return `ok(transactions)`.

**Route 4: `src/app/api/inventory/[id]/checkout/route.ts`**
- `POST`: Permission `INVENTORY_CHECKOUT`. Parse body, validate with `CheckoutSchema.safeParse`. Call `checkoutItem(orgId, id, validated.data, ctx.userId)`. Catch INSUFFICIENT_STOCK -> return `fail('INSUFFICIENT_STOCK', 'Not enough items in stock')` with status 409. Catch NOT_FOUND -> 404.

**Route 5: `src/app/api/inventory/[id]/checkin/route.ts`**
- `POST`: Permission `INVENTORY_CHECKIN`. Parse body, validate with `CheckinSchema.safeParse`. Call `checkinItem(orgId, validated.data, ctx.userId)`. Catch ALREADY_CHECKED_IN -> return `fail('ALREADY_CHECKED_IN', 'This item has already been checked in')` with status 409. Catch NOT_FOUND -> 404.

Error handling pattern for service errors: The service functions throw `new Error('NOT_FOUND')`, `new Error('INSUFFICIENT_STOCK')`, `new Error('ALREADY_CHECKED_IN')`. In the catch block, check `error.message` and map to the appropriate HTTP status code and error code.
