1. **Input sanitization** (`src/lib/sanitize.ts`):
   - Create a lightweight HTML sanitizer that strips dangerous content from user input. Do NOT add a heavy dependency like DOMPurify (adds ~60KB and requires DOM shim for server).
   - Implement `sanitizeHtml(input: string): string` that:
     - Strips `<script>...</script>` tags and their content (case-insensitive, handles attributes)
     - Strips `<style>...</style>` tags and their content
     - Strips event handler attributes: `on\w+="..."` patterns (onclick, onerror, onload, etc.)
     - Strips `javascript:` protocol in href/src attributes
     - Strips `<iframe>`, `<embed>`, `<object>`, `<form>`, `<input>` tags
     - Preserves safe HTML tags for rich text fields: `<p>`, `<br>`, `<b>`, `<i>`, `<strong>`, `<em>`, `<ul>`, `<ol>`, `<li>`, `<a>` (with href), `<h1>`-`<h6>`
     - For plain text fields, provide `stripAllHtml(input: string): string` that strips ALL tags
   - Create `zodSanitizedString()` factory that returns `z.string().transform(stripAllHtml)` — for use in Zod schemas
   - Create `zodRichText()` factory that returns `z.string().transform(sanitizeHtml)` — for rich text fields
   - Export all: `sanitizeHtml`, `stripAllHtml`, `zodSanitizedString`, `zodRichText`

2. **File upload validation** (`src/lib/validation/file-upload.ts`):
   - Define allowed MIME type constants:
     ```typescript
     export const ALLOWED_IMAGE_TYPES = new Set(['image/jpeg', 'image/png', 'image/webp', 'image/gif'])
     export const ALLOWED_DOCUMENT_TYPES = new Set([...ALLOWED_IMAGE_TYPES, 'application/pdf'])
     export const MAX_FILE_SIZE_BYTES = 10 * 1024 * 1024 // 10MB
     ```
   - Create `validateFileUpload(file: { type: string; size: number; name: string }, options?: { allowedTypes?: Set<string>; maxSizeBytes?: number }): { valid: boolean; error?: string }`:
     - Check MIME type against allowlist (default: ALLOWED_IMAGE_TYPES)
     - Check file size against limit (default: MAX_FILE_SIZE_BYTES)
     - Return descriptive error: `"File type '${file.type}' is not allowed. Accepted: JPEG, PNG, WebP, GIF"` or `"File '${file.name}' exceeds the 10MB size limit"`
   - Create `validateMultipleFiles(files: Array<{ type: string; size: number; name: string }>, options?)` that validates each file and returns aggregated result
   - Export all constants and functions

   - `verifyHmacSha256(payload: string, signature: string, secret: string): boolean`:
     - Compute HMAC-SHA256 of payload using secret
     - Compare with provided signature using `crypto.timingSafeEqual` to prevent timing attacks
     - Handle hex-encoded signatures (most common)
     - Return boolean
   - Export the function

2. **Stripe webhook** (`src/app/api/platform/webhooks/stripe/route.ts`):
   - Read the existing file first to understand its structure
   - Add signature verification at the TOP of the POST handler, before any payload processing:
     ```typescript
     const signature = req.headers.get('stripe-signature')
     if (!signature) {
       return NextResponse.json(fail('UNAUTHORIZED', 'Missing webhook signature'), { status: 401 })
     }
     const rawBody = await req.text()
     const stripeWebhookSecret = process.env.STRIPE_WEBHOOK_SECRET
     if (!stripeWebhookSecret) {
       console.error('[Stripe webhook] STRIPE_WEBHOOK_SECRET not configured')
       return NextResponse.json(fail('INTERNAL_ERROR', 'Webhook not configured'), { status: 500 })
     }
     ```
   - If the project uses the `stripe` npm package: use `stripe.webhooks.constructEvent(rawBody, signature, stripeWebhookSecret)` which handles verification + parsing
   - If the `stripe` package is not installed: use the shared `verifyHmacSha256` helper with Stripe's signature format (Stripe uses `t=timestamp,v1=signature` format — parse the `v1` component and compute HMAC of `${timestamp}.${rawBody}`)
   - Catch verification errors and return 401

3. **Clever webhook** (`src/app/api/webhooks/clever/route.ts`):
   - Add at the TOP of POST handler, before body parsing:
     ```typescript
     const signature = req.headers.get('x-clever-signature') || req.headers.get('clever-signature')
     const cleverSecret = process.env.CLEVER_WEBHOOK_SECRET
     if (!cleverSecret) {
       // If no secret configured, log warning and skip verification (graceful degradation)
       console.warn('[Clever webhook] CLEVER_WEBHOOK_SECRET not configured — skipping signature verification')
     } else if (!signature) {
       return NextResponse.json(fail('UNAUTHORIZED', 'Missing webhook signature'), { status: 401 })
     } else {
       const rawBody = await req.text()
       const isValid = verifyHmacSha256(rawBody, signature, cleverSecret)
       if (!isValid) {
         return NextResponse.json(fail('UNAUTHORIZED', 'Invalid webhook signature'), { status: 401 })
       }
       // Re-parse body since we consumed it
       const body = JSON.parse(rawBody)
       // ... continue with existing logic using body
     }
     ```
