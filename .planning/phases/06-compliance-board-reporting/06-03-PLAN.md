FCI Formula:
```
FCI = sum(estimatedRepairCostUSD for BACKLOG/TODO/IN_PROGRESS/ON_HOLD tickets) / sum(replacementCost for all ACTIVE assets)
FCI < 0.05 = Good (green), 0.05-0.10 = Fair (amber), > 0.10 = Poor (red)
```

1. **boardReportService.ts** — core analytics and export logic:

Exported types:
```typescript
export type BoardReportMetrics = {
  period: { from: Date; to: Date }
  fci: { score: number; deferred: number; replacementValue: number; rating: 'GOOD' | 'FAIR' | 'POOR' }
  costPerStudent: number | null
  pmVsReactiveRatio: { pmCount: number; reactiveCount: number; pmPct: number }
  deferredBacklog: { count: number; estimatedCostUSD: number }
  responseTime: { avgHours: number; byCategory: Record<string, number>; byCampus: Record<string, number> }
  resolutionTime: { avgHours: number; byCategory: Record<string, number>; byCampus: Record<string, number> }
  complianceStatus: { byDomain: Record<string, { total: number; current: number; overdue: number; pct: number }> }
  assetEOLForecast: { in1Year: number; in3Years: number; in5Years: number; totalReplacementCost1: number; totalReplacementCost3: number; totalReplacementCost5: number }
  topRepairCostAssets: Array<{ assetId: string; assetNumber: string; name: string; cumulativeRepairCost: number; replacementCost: number | null; repairPct: number }>
  yoyComparison: { thisYear: { ticketCount: number; totalCost: number; avgResolutionHours: number }; lastYear: { ticketCount: number; totalCost: number; avgResolutionHours: number } }
}
```

Exported functions:

`calculateFCI(orgId: string)`:
- Use rawPrisma (cross-org not needed, but rawPrisma for explicit queries)
- Get all ACTIVE MaintenanceAssets with replacementCost set → sum replacementCost = `totalReplacementValue`
- Get all MaintenanceTickets with status IN (BACKLOG, TODO, IN_PROGRESS, ON_HOLD) and estimatedRepairCostUSD set → sum = `deferredMaintenance`
- FCI score = deferredMaintenance / totalReplacementValue (return 0 if no assets)
- Return `{ score, deferred: deferredMaintenance, replacementValue: totalReplacementValue, rating }`

`getBoardReportMetrics(orgId: string, filters: { from: Date; to: Date; schoolId?: string })`:
- Run all metric queries in parallel using Promise.all for performance
- Query group 1 — ticket metrics (filter by organizationId + createdAt BETWEEN from/to + optional schoolId):
  - PM vs reactive: count tickets where pmScheduleId IS NOT NULL (PM) vs null (reactive)
  - Deferred backlog: count + sum(estimatedRepairCostUSD) for open tickets
  - Response time: for DONE/QA tickets, compute hours between createdAt and first IN_PROGRESS activity. Use rawPrisma to join MaintenanceTicketActivity where type='STATUS_CHANGE' AND toStatus='IN_PROGRESS'. Group by category and school/campus.
  - Resolution time: hours between createdAt and status=DONE. Group by category and school/campus.
- Query group 2 — assets:
  - calculateFCI(orgId)
  - EOL forecast: assets where (createdAt year + expectedLifespanYears) falls within 1/3/5 years from now
  - Top repair cost: join MaintenanceCostEntry + MaintenanceLaborEntry × TechnicianProfile.loadedHourlyRate per asset. Top 10 by cumulative cost. Include repair cost vs replacement cost ratio.
- Query group 3 — compliance (from ComplianceRecord):
  - Group by domain, count total/current/overdue
- Query group 4 — org info:
  - Organization.studentCount for cost per student
  - Sum all MaintenanceCostEntry.amount + MaintenanceLaborEntry labor cost in period = total cost
  - costPerStudent = totalCost / studentCount (null if studentCount null or 0)
- Query group 5 — YoY: repeat ticket count/cost/resolution for same date range one year prior

`generateAINarrative(metrics: BoardReportMetrics, orgName: string)`:
- Construct a detailed prompt:
  ```
  You are writing an executive narrative summary for a K-12 school board facilities report.
  Be professional, concise (3-4 paragraphs), and highlight key findings and recommendations.
  The tone should be informative but accessible to non-technical board members.

  Facilities Data:
  - FCI Score: {fci.score.toFixed(3)} ({fci.rating}) — {fci.rating === 'GOOD' ? 'facility is in good condition' : fci.rating === 'FAIR' ? 'attention recommended' : 'immediate action required'}
  - Deferred maintenance backlog: {count} tickets, estimated ${cost}
  - PM vs Reactive ratio: {pmPct}% preventive maintenance
  - Cost per student: ${costPerStudent}
  - Compliance: {summary of current/overdue by domain}
  - Assets reaching end of life: {in1Year} in next year, {in3Years} in 3 years
  - Top repair cost asset: {name} at ${cost} cumulative
  - Year-over-year: ticket volume {up/down X%}, avg resolution {faster/slower}

  Write a 3-4 paragraph executive narrative.
  ```
- Call Anthropic claude-sonnet-4-5 with max_tokens: 1024
- Return narrative string. If ANTHROPIC_API_KEY not set or call fails, return a fallback template string with the key metrics embedded.

`exportBoardReportPDF(orgId: string, orgName: string, metrics: BoardReportMetrics, narrative: string)`:
- Build PDF using jsPDF:
  - Page 1: Cover — "Facilities Board Report", org name, period, FCI score (large, color-coded), generated timestamp
  - Page 2: Executive Summary — AI narrative text, word-wrapped using `doc.splitTextToSize()`
  - Page 3: Key Metrics — 2-column layout: FCI gauge info, cost per student, PM ratio, backlog count/cost, response/resolution averages
  - Page 4: Compliance Status — table of 10 domains with % current column
  - Page 5: Asset Intelligence — EOL forecast table + top 10 repair cost assets table
  - Page 6: Year-over-Year Comparison — this year vs last year for ticket count, total cost, avg resolution
  - Footer each page: org name + "Lionheart Facilities Management" + page N of M
- Return Buffer from `doc.output('arraybuffer')`
