Output: Ticket auto-generation API, document upload API, jsPDF audit export endpoint, and the compliance record editing drawer with attachment panel and export dialog.

1. **Add to complianceService.ts** — three new exported functions:

`generateComplianceTicket(orgId, recordId, submittedByUserId)`:
- Load ComplianceRecord by id (asserting it belongs to orgId)
- If record.generatedTicketId already set, throw "Ticket already generated for this compliance record"
- Look up COMPLIANCE_DOMAIN_DEFAULTS[record.domain] for category mapping
- Call `generateTicketNumber(orgId)` from maintenanceTicketService.ts
- Create MaintenanceTicket via rawPrisma:
  ```typescript
  {
    organizationId: orgId,
    ticketNumber,
    title: `Compliance: ${COMPLIANCE_DOMAIN_DEFAULTS[record.domain].label} – ${record.title}`,
    description: `Auto-generated from compliance record. Due: ${record.dueDate.toLocaleDateString()}`,
    status: 'BACKLOG',
    category: COMPLIANCE_DOMAIN_DEFAULTS[record.domain].category,
    specialty: categoryToSpecialtyMap[COMPLIANCE_DOMAIN_DEFAULTS[record.domain].category],
    priority: daysUntilDue < 14 ? 'URGENT' : daysUntilDue < 30 ? 'HIGH' : 'MEDIUM',
    submittedById: submittedByUserId,
    schoolId: record.schoolId,
    version: 1,
  }
  ```
- Update ComplianceRecord to set `generatedTicketId` to the new ticket's id
- Return the created ticket

`generateRemediationTicket(orgId, recordId, submittedByUserId)`:
- Mirror generateComplianceTicket but:
  - Title prefix: `Remediation Required: ${domainLabel}`
  - Priority: always URGENT
  - Description includes "FAILED inspection on {inspectionDate}. Remediation required."
  - Status: BACKLOG
- Only callable when record.outcome === 'FAILED'
- Store a separate `remediationTicketId` field — NOTE: add `remediationTicketId String?` to ComplianceRecord model in prisma/schema.prisma and run `npm run db:push` again. Add the back-relation to MaintenanceTicket as `complianceRemediationRecord ComplianceRecord? @relation("RemediationTicket", fields: ..., references: ...)`. Name the relation "RemediationTicket" to avoid collision with "ComplianceTicket".

`getComplianceRecordsForExport(orgId, filters: { from: Date; to: Date; schoolId?: string; domain?: ComplianceDomain })`:
- Query all ComplianceRecords (including soft-deleted = false) for the period
- Include domainConfig, school, generatedTicket relations
- Return array sorted by domain ASC, dueDate ASC

2. **POST /api/maintenance/compliance/records/[id]/generate-ticket**:
- Permission: COMPLIANCE_MANAGE
- Body: optional `{ type: 'compliance' | 'remediation' }` (default 'compliance')
- If type is 'remediation': validate that record.outcome === 'FAILED', else return 400 "Record must have FAILED outcome for remediation ticket"
- Call generateComplianceTicket or generateRemediationTicket
- Return `ok({ ticket: createdTicket, record: updatedRecord })`

3. **POST /api/maintenance/compliance/records/[id]/upload-url**:
- Permission: COMPLIANCE_MANAGE
- Body: `{ filename: string; contentType: string }`
- Validate content type is image/* or application/pdf or application/msword or similar document type
- Generate Supabase signed upload URL for bucket `compliance-docs` with path `{orgId}/{recordId}/{timestamp}-{filename}`
- Return `ok({ uploadUrl, fileUrl })`
- After upload, client should call PATCH /api/maintenance/compliance/records/[id] to append fileUrl to attachments array

4. **GET /api/maintenance/compliance/export**:
- Permission: COMPLIANCE_EXPORT
- Query params: `?from=YYYY-MM-DD&to=YYYY-MM-DD&schoolId=optional&domain=optional`
- Validate from/to date params with Zod (required)
- Call getComplianceRecordsForExport(orgId, { from, to, schoolId, domain })
- Build PDF using jsPDF:
  - Page 1 header: Organization name, "Compliance Audit Report", date range, generated timestamp
  - Summary table: 10 rows (one per domain) — Domain | Records | Current | Overdue | Last Inspection
  - Detail section: for each domain, a section header + records table (Title | Due Date | Inspection Date | Outcome | Inspector | Attached Docs count)
  - Footer on each page: "Generated by Lionheart Facilities Management — Page N of M" using `doc.setPage()` loop
  - Font sizes: 18pt header, 12pt section headers, 10pt body text
  - Use Helvetica (built-in), no external font loading needed
  - AVOID jspdf-autotable dependency (not installed) — build tables manually with doc.text() + doc.line() or use a simple loop
- Install jspdf if not in package.json: `npm install jspdf`
- Return Response with Content-Type application/pdf, Content-Disposition attachment filename=`compliance-audit-{YYYY-MM-DD}.pdf`
- NOTE: This is a Next.js route handler — do NOT use `res.send()`. Use `return new Response(pdfBuffer, { headers: {...} })` directly.
