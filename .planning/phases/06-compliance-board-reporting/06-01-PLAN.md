```prisma
enum ComplianceDomain {
  AHERA          // Asbestos — annual inspection + 3-year re-inspection
  FIRE_SAFETY    // Annual fire inspection + monthly extinguisher checks
  PLAYGROUND     // Annual playground safety inspection
  LEAD_WATER     // Lead in water testing — every 3 years
  BOILER         // Annual boiler inspection
  ELEVATOR       // Annual elevator inspection (if applicable)
  KITCHEN        // Annual health/sanitation inspection
  ADA            // ADA self-evaluation — every 3 years
  RADON          // Radon testing — every 5 years
  IPM            // Integrated Pest Management annual review
}

enum ComplianceStatus {
  CURRENT        // Inspection complete, within valid period
  DUE_SOON       // Within 30 days of deadline
  OVERDUE        // Past deadline with no passing record
  NOT_APPLICABLE // Domain disabled for this school
  PENDING        // Deadline exists but not yet inspected
}

enum ComplianceOutcome {
  PASSED
  FAILED
  CONDITIONAL_PASS
  PENDING
}
```

Add models:
```prisma
model ComplianceDomainConfig {
  id             String          @id @default(cuid())
  organizationId String
  schoolId       String?         // null = org-wide config
  domain         ComplianceDomain
  isEnabled      Boolean         @default(true)
  customDeadlineMonth Int?       // Override: 1-12, null uses domain default
  customDeadlineDay   Int?       // Override: 1-31, null uses domain default
  notes          String?
  createdAt      DateTime        @default(now())
  updatedAt      DateTime        @updatedAt

  organization Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  school       School?      @relation(fields: [schoolId], references: [id], onDelete: SetNull)
  records      ComplianceRecord[]

  @@unique([organizationId, schoolId, domain])
  @@index([organizationId])
  @@index([schoolId])
}

model ComplianceRecord {
  id                    String           @id @default(cuid())
  organizationId        String
  domainConfigId        String
  schoolId              String?
  domain                ComplianceDomain
  title                 String           // e.g. "Annual Fire Safety Inspection 2026"
  dueDate               DateTime
  inspectionDate        DateTime?
  outcome               ComplianceOutcome @default(PENDING)
  status                ComplianceStatus  @default(PENDING)
  inspector             String?          // Inspector name/company
  notes                 String?
  attachments           String[]         @default([])  // Supabase signed URLs
  generatedTicketId     String?          // FK to MaintenanceTicket auto-created for this
  remindedAt30Days      Boolean          @default(false)
  remindedAt7Days       Boolean          @default(false)
  deletedAt             DateTime?
  createdAt             DateTime         @default(now())
  updatedAt             DateTime         @updatedAt

  organization  Organization          @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  domainConfig  ComplianceDomainConfig @relation(fields: [domainConfigId], references: [id], onDelete: Cascade)
  school        School?               @relation(fields: [schoolId], references: [id], onDelete: SetNull)
  generatedTicket MaintenanceTicket?  @relation("ComplianceTicket", fields: [generatedTicketId], references: [id], onDelete: SetNull)

  @@index([organizationId, domain])
  @@index([organizationId, dueDate])
  @@index([organizationId, status])
  @@index([domainConfigId])
  @@index([organizationId, deletedAt])
}
```

4. **Compliance service** (`src/lib/services/complianceService.ts`):

Domain default schedule — define a constant `COMPLIANCE_DOMAIN_DEFAULTS` mapping each domain to:
- `label`: Human-readable name (e.g., "AHERA Asbestos Inspection")
- `description`: Brief regulatory description
- `defaultMonth`: Default due month (1-12)
- `defaultDay`: Default due day (1-31)
- `frequencyYears`: How often required (1 = annual, 3 = every 3 years, 5 = every 5 years)
- `category`: MaintenanceCategory for auto-generated tickets

Domain defaults:
- AHERA: month 9, day 1, annual (STRUCTURAL category)
- FIRE_SAFETY: month 9, day 15, annual (STRUCTURAL)
- PLAYGROUND: month 8, day 15, annual (GROUNDS)
- LEAD_WATER: month 6, day 1, every 3 years (PLUMBING)
- BOILER: month 10, day 1, annual (HVAC)
- ELEVATOR: month 10, day 15, annual (STRUCTURAL)
- KITCHEN: month 8, day 1, annual (CUSTODIAL_BIOHAZARD)
- ADA: month 3, day 1, every 3 years (STRUCTURAL)
- RADON: month 11, day 1, every 5 years (OTHER)
- IPM: month 8, day 31, annual (GROUNDS)

Exported functions:
- `COMPLIANCE_DOMAINS` constant — array of all 10 domain keys
- `createComplianceDomainConfig(orgId, input)` — upsert domain config for org+school+domain combo. Validates with Zod.
- `getComplianceDomainConfigs(orgId, schoolId?)` — list all 10 domains showing config (enabled/disabled) for the given school. Returns all 10 regardless of whether a config row exists (defaults to enabled for all).
- `updateComplianceDomainConfig(orgId, id, data)` — partial update
- `populateComplianceCalendar(orgId, schoolYearStart: Date, schoolYearEnd: Date)` — for each enabled domain config, compute the due dates in the year range using frequencyYears logic (skip if last record for domain is not yet expired). Create ComplianceRecord rows via `prisma.complianceRecord.upsert` keyed on `(organizationId, domainConfigId, dueDate)` to ensure idempotency.
- `getComplianceRecords(orgId, filters?)` — list with optional domain/status/dateRange filters, include domainConfig relation
- `getComplianceRecordById(orgId, id)` — full record with relations
- `updateComplianceRecord(orgId, id, data)` — partial update (outcome, inspectionDate, inspector, notes, attachments)
- `sendComplianceReminders(orgId?)` — used by cron. If orgId provided, process just that org; otherwise process ALL orgs via rawPrisma. For each ComplianceRecord where dueDate is within 30 days AND remindedAt30Days=false (or within 7 days AND remindedAt7Days=false): find Head and Admin users in the org (users with MAINTENANCE_READ_ALL or admin role), send email via emailService `sendComplianceReminderEmail()`, create Notification rows, mark the reminder flag.

6. **API routes** (all follow standard getOrgIdFromRequest → getUserContext → assertCan → runWithOrgContext pattern):

- `GET /api/maintenance/compliance/domains` — list 10 domain configs for org. Permission: COMPLIANCE_READ
- `POST /api/maintenance/compliance/domains` — create/update domain config (body: `{ domain, isEnabled, customDeadlineMonth?, customDeadlineDay?, schoolId?, notes? }`). Permission: COMPLIANCE_MANAGE. After updating, call `populateComplianceCalendar` for the current school year.
- `GET /api/maintenance/compliance/domains/[id]` — get single config. Permission: COMPLIANCE_READ
- `PATCH /api/maintenance/compliance/domains/[id]` — update config. Permission: COMPLIANCE_MANAGE
- `GET /api/maintenance/compliance/records` — list records with filters (?domain=, ?status=, ?schoolId=, ?from=, ?to=). Permission: COMPLIANCE_READ
- `GET /api/maintenance/compliance/records/[id]` — detail. Permission: COMPLIANCE_READ
- `PATCH /api/maintenance/compliance/records/[id]` — update outcome/inspectionDate/inspector/notes/attachments. Permission: COMPLIANCE_MANAGE
- `POST /api/maintenance/compliance/records/upload-url` — signed URL for compliance document attachments (bucket: `compliance-docs`). Permission: COMPLIANCE_MANAGE
