**1b. SubmitRequestWizard.tsx** — Orchestrator component:
- 4-step wizard with progress bar at top showing: Location, Photos, Details, Review
- Progress bar shows current step highlighted (emerald-500), completed steps with checkmarks, future steps grayed
- Uses `useState` for currentStep (0-3) and form data object: `{ buildingId, areaId, roomId, locationLabel, photos: {url, fileName}[], title, description, category, priority, availabilityNote, scheduledDate, aiSuggestedCategory, aiMultiIssue }`
- Step transitions use Framer Motion `AnimatePresence` with fadeInUp variant for forward, reverse for backward
- "Next" button disabled until step validation passes (Location: roomId required; Photos: no validation - optional; Details: title required + category required; Review: always valid)
- "Back" button on steps 2-4, "Cancel" button always
- On final submit: POST /api/maintenance/tickets with form data. Show emerald success state with ticket number (MT-XXXX) and "View Ticket" link to `/maintenance/tickets/[id]`
- Render as a full-page overlay or panel within MyRequestsView (not a modal — it's the main content area when active)

**1c. StepLocation.tsx** — Location search + picker:
- Search input with placeholder "Search rooms, areas, or buildings..."
- Use `useDeferredValue` (React 18) for debounced filtering of the flat location list from useCampusLocations
- Results appear in a dropdown below the input, grouped by building name
- Each result shows full hierarchy path: "Building > Area > Room" with room name/code prominent
- Clicking a result sets buildingId, areaId, roomId, and locationLabel in wizard state
- Selected location shown as a card below the search input (ui-glass-hover with building/area/room chips)
- "Clear" button to re-select
- If no campus data loaded yet, show loading skeleton

**1d. StepPhotos.tsx** — Photo upload:
- "Add Photo" button that opens native file input (accept="image/*" capture="environment" for mobile camera)
- Also supports drag-and-drop for desktop (adapt ImageDropZone pattern)
- Up to 5 photos. Counter shows "X/5 photos"
- Upload flow: (1) select file -> (2) POST /api/maintenance/tickets/upload-url to get signed URL -> (3) PUT file directly to Supabase via signed URL -> (4) store publicUrl in wizard state
- Show upload progress per photo (uploading spinner, done checkmark)
- Thumbnails in a grid (2x3) with remove button (X) overlay
- After first photo upload completes, fire AI category suggestion in background: POST /api/maintenance/tickets/ai-suggest-category with base64 of first photo. Store result in wizard state as aiSuggestedCategory. Do NOT block the user — they can proceed immediately
- If AI fails, silently ignore (suggestedCategory stays null)
- File type validation: only images (jpg, png, webp, heic). Max 10MB per file

**1e. StepDetails.tsx** — Title, category, priority, description:
- Title input (required, max 200 chars, counter showing remaining)
- Category dropdown (8 options from MaintenanceCategory enum). If aiSuggestedCategory is set, pre-select it and show subtle "AI suggested" chip next to the dropdown. User can freely change
- Priority radio buttons (Low / Medium / High / Urgent) with color-coded backgrounds. Default: Medium
- Description textarea (optional, max 2000 chars)
- Availability note input (optional — "When can the maintenance team access this room?")
- "Schedule for later" toggle that reveals a date picker when enabled. Date must be in the future. This sets scheduledDate in wizard state (ticket enters SCHEDULED instead of BACKLOG)

**1f. StepReview.tsx** — Summary + AI multi-issue + submit:
- Full summary card showing: Location (hierarchy), Photos (thumbnails), Title, Category, Priority, Description, Availability, Scheduled date (if set)
- On mount (or when this step becomes active), fire AI multi-issue detection: POST /api/maintenance/tickets/ai-detect-multi-issue with { title, description, category }. If hasMultipleIssues, show amber banner: "This looks like it might describe two separate issues. Splitting into separate tickets helps the maintenance team address each one faster." with two buttons: "Split into 2 tickets" and "Submit as one ticket anyway"
- "Split into 2 tickets" flow: submit first ticket as-is. Then reset wizard with same location pre-filled, AI-suggested title and category for second issue. Show "First ticket MT-XXXX created. Now complete the second ticket."
- "Submit" button (emerald, prominent). Shows loading spinner during POST. On success: show success state with ticket number, "View Ticket" link, "Submit Another" button
- On API error: show toast with error message, keep form data intact

All UI must follow glassmorphism system: ui-glass for cards, ui-glass-overlay for the wizard container, input classes per convention. Mobile-first: wizard fills available space, large touch targets (min 44px), comfortable spacing.
