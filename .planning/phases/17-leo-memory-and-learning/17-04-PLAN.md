Create `src/lib/services/ai/memoryExtractionService.ts`:

**extractMemoryFromConversation(conversationId: string, userId: string): Promise<void>**
This is the main post-conversation extraction pipeline. It runs as a fire-and-forget background task.

1. Fetch all messages for the conversation via `getMessages(conversationId, { limit: 100 })`
2. Filter to only user and assistant messages (skip tool_call/tool_result)
3. If fewer than 3 messages, skip extraction (not enough signal)
4. Build a conversation transcript: join messages as "User: ...\nAssistant: ..."
5. Call Gemini to extract structured data:

```
const prompt = `Analyze this conversation between a user and an AI assistant at a school operations platform. Extract the following in JSON format:

{
  "facts": [
    { "text": "specific fact about the user or their work", "category": "preference|pattern|context|instruction", "importance": 0.0-1.0 }
  ],
  "profile_updates": {
    "frequent_topics": ["topic1", "topic2"],
    "common_actions": ["action1"],
    "domain_expertise": ["area1"],
    "tone_preference": "casual|professional|mixed|null",
    "response_length": "short|medium|detailed|null",
    "communication_style_notes": "any notes about how they communicate"
  }
}

Rules:
- Only extract CONCRETE facts, not generic observations
- Facts should be about the user's role, preferences, common tasks, or institutional knowledge
- Importance: 0.9+ for explicit instructions ("always do X"), 0.5-0.8 for preferences, 0.3-0.5 for contextual
- Skip facts that are already obvious from the user's role
- Return empty arrays/null if nothing notable

Conversation:
${transcript}`
```

6. Parse the Gemini response as JSON (with try/catch for malformed responses)
7. For each extracted fact:
   - Check if a similar fact already exists (query UserMemoryFact by userId and category, compare text similarity)
   - If new, create UserMemoryFact with `rawPrisma.userMemoryFact.create(...)` and then call `generateAndStoreEmbedding('UserMemoryFact', factId, factText)` for the embedding
   - If similar exists (>80% string overlap), update importance (take the higher value)
8. Call `updateUserProfile(userId, profileUpdates)`

**updateUserProfile(userId: string, updates: ProfileUpdates): Promise<void>**
1. Upsert UserAssistantProfile using rawPrisma
2. For array fields (frequentTopics, commonActions, domainExpertise): merge with existing, deduplicate, keep most recent 20 items
3. For scalar fields (responseLength, tonePreference): only overwrite if the new value is non-null
4. Increment conversationCount by 1
5. Update lastUpdated timestamp

**decayMemoryImportance(userId: string): Promise<void>**
Utility for future use — reduces importance of all UserMemoryFacts by 5% (multiply by 0.95). Called optionally during extraction. Facts below 0.1 importance can be pruned later.
- `rawPrisma.$executeRawUnsafe('UPDATE "UserMemoryFact" SET importance = importance * 0.95 WHERE "userId" = $1 AND importance > 0.1', userId)`

All functions use Pino logger for structured logging. Errors are caught and logged but never thrown (this runs in background).

**Create `src/lib/services/ai/contextAssemblyService.ts`:**

**assembleContext(userId: string, orgId: string, currentMessage: string): Promise<AssembledContext>**

Returns:
```typescript
interface AssembledContext {
  userProfile: { responseLength?: string, tonePreference?: string, frequentTopics: string[], domainExpertise: string[], communicationStyle?: any } | null
  relevantFacts: Array<{ factText: string, category: string, importance: number }>
  recentSummaries: Array<{ summaryText: string, conversationTitle?: string }>
}
```

Implementation:
1. **Load user profile:** `rawPrisma.userAssistantProfile.findUnique({ where: { userId } })` — if null, return empty profile
2. **Load relevant memory facts:**
   - Generate embedding for `currentMessage`
   - Search UserMemoryFact table: `searchSimilar('UserMemoryFact', queryVec, { limit: 10, orgId: null })` with filter `AND "userId" = '${userId}'`
   - Sort by (similarity * importance) to favor high-importance relevant facts
   - Take top 5
   - Fetch full records to get factText, category, importance
3. **Load recent conversation summaries:**
   - Query ConversationSummary joined with Conversation where userId matches, ordered by createdAt desc, limit 3
   - Return summaryText and conversation title
4. Return all assembled context

If Gemini API key is missing (no embedding possible), fall back to:
- Still load user profile (no embedding needed)
- Load top 5 most important UserMemoryFacts by importance score (no semantic search)
- Load recent summaries normally
