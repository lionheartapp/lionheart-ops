**Create `src/lib/services/ai/conversationSummarizationService.ts`:**

**shouldSummarize(conversationId: string): Promise<boolean>**
- Count messages in the conversation via rawPrisma.conversationMessage.count({ where: { conversationId } })
- Check if a summary already exists for this conversation that covers most messages
- Return true if message count >= 20 AND (no summary exists OR summary.messageCount < current count - 10)
- The "- 10" buffer means we don't re-summarize after every single new message

**summarizeConversation(conversationId: string): Promise<void>**
1. Fetch all messages: `getMessages(conversationId, { limit: 200 })`
2. Separate into "old" (first 75% of messages) and "recent" (last 25%)
3. Build transcript of the OLD messages only (the recent ones stay as verbatim context)
4. Call Gemini to summarize:

```
const prompt = `Summarize this conversation between a user and Leo (an AI assistant at a school operations platform). Focus on:
1. Key topics discussed
2. Decisions made or actions taken
3. Important context or facts mentioned
4. Any unresolved questions or pending items

Keep the summary concise (200-300 words). Write in third person: "The user asked about..." not "You asked about..."

Conversation:
${transcript}`
```

5. Create a ConversationSummary record via rawPrisma:
   ```
   rawPrisma.conversationSummary.create({
     data: {
       conversationId,
       summaryText: summary,
       messageCount: oldMessages.length,
       tokenCount: estimateTokenCount(summary),  // rough estimate: chars / 4
     }
   })
   ```
6. Log success with Pino logger

**estimateTokenCount(text: string): number**
- Simple heuristic: `Math.ceil(text.length / 4)` — good enough for budgeting

All operations wrapped in try/catch, errors logged but not thrown.
