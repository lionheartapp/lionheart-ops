**Create `src/lib/services/ai/tools/memory.tools.ts`:**

Register a `recall_context` tool via `registerTools()` (self-registration pattern matching other tools):

Definition:
```
name: 'recall_context'
description: 'Search organizational history using semantic similarity. Use this when the user asks about past events, previous tickets, patterns, or "what happened last time". Returns the most relevant records from the organization\'s data.'
parameters:
  type: 'object'
  properties:
    query: { type: 'string', description: 'The search query describing what to find' }
    search_scope: { type: 'string', enum: ['tickets', 'events', 'inventory', 'conversations', 'all'], description: 'Which data to search. Default: all' }
    time_range: { type: 'string', enum: ['week', 'month', 'quarter', 'year', 'all'], description: 'How far back to search. Default: all' }
    limit: { type: 'number', description: 'Max results to return. Default: 5' }
  required: ['query']
```

requiredPermission: null (all users can search within their org scope)
riskTier: 'GREEN' (read-only)

Execute function:
1. Call `generateEmbedding(input.query as string)` to get query vector
2. If empty (no API key), return `{ error: 'Semantic search unavailable' }`
3. Based on search_scope (default 'all'), search across relevant tables:
   - 'tickets' or 'all': `searchSimilar('Ticket', queryVec, { limit, orgId: ctx.organizationId })`
   - 'events' or 'all': `searchSimilar('"CalendarEvent"', queryVec, { limit, orgId: ctx.organizationId })`
   - 'inventory' or 'all': `searchSimilar('"InventoryItem"', queryVec, { limit, orgId: ctx.organizationId })`
   - 'conversations' or 'all': `searchSimilar('"ConversationMessage"', queryVec, { limit, orgId: ctx.organizationId })`
4. For time_range filtering, build a date filter: `AND "createdAt" > '${cutoffDate.toISOString()}'` and pass as `filters` parameter
5. Merge all results, sort by similarity, take top `limit` (default 5)
6. For each result, fetch the actual record details using rawPrisma:
   - Tickets: title, description, status, category, priority, createdAt
   - CalendarEvent: title, description, startTime, locationText
   - InventoryItem: name, description, category, quantityOnHand
   - ConversationMessage: content, role, createdAt (with conversation title)
7. Return JSON string with results array: `{ results: [...], total: N, scope: search_scope }`

**Note on table names:** Prisma table names are PascalCase by default. The searchSimilar function needs the actual PostgreSQL table name. Check if Prisma uses the model name directly — it does by default (e.g., `Ticket`, `CalendarEvent`). Use double-quotes in SQL for case-sensitive table names.

**Create `src/lib/services/ai/embeddingTriggers.ts`:**

Three functions that build a text representation and call generateAndStoreEmbedding:

`embedTicket(ticketId: string, ticket: { title: string, description?: string | null, category?: string, priority?: string }): void`
- Build text: `${ticket.title} ${ticket.description || ''} ${ticket.category || ''} ${ticket.priority || ''}`
- Call `void generateAndStoreEmbedding('Ticket', ticketId, text)` (fire-and-forget, wrapped in try/catch)

`embedCalendarEvent(eventId: string, event: { title: string, description?: string | null, locationText?: string | null }): void`
- Build text: `${event.title} ${event.description || ''} ${event.locationText || ''}`
- Fire-and-forget

`embedInventoryItem(itemId: string, item: { name: string, description?: string | null, category?: string | null }): void`
- Build text: `${item.name} ${item.description || ''} ${item.category || ''}`
- Fire-and-forget

All three are sync functions that internally call async operations with `void`. They catch and log errors with Pino logger but never throw.
