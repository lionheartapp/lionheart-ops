<objective>
Fix two API response data-shape parsing bugs that prevent conversation history from displaying in Leo's UI.

Purpose: Both `GET /api/conversations` and `GET /api/conversations/[id]/messages` wrap their arrays inside named properties (`data.conversations` and `data.messages` respectively), but the frontend components parse `json.data` as if it were a direct array. This makes the conversation sidebar always show "No conversations yet" and loading a past conversation silently fail. These are the only two gaps blocking LEO-MEM-01 from being fully functional.

Output: Two corrected components that properly destructure the API response envelope.
</objective>
