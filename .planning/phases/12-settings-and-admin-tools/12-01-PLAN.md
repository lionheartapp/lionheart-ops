**First, add server-side date range filtering to the existing audit-logs API route.**

Update `src/app/api/settings/audit-logs/route.ts` to support `from` and `to` query params for date range filtering. This is required because the API is paginated — client-side date filtering would only filter the current page, not the full dataset.

Add after the existing `resourceType` param parsing:
```typescript
const from = searchParams.get('from') ?? undefined  // ISO date string
const to   = searchParams.get('to')   ?? undefined  // ISO date string

// In the where clause:
if (from || to) {
  where.createdAt = {}
  if (from) (where.createdAt as Record<string, unknown>).gte = new Date(from)
  if (to)   (where.createdAt as Record<string, unknown>).lte = new Date(to)
}
```

This ensures date filtering happens at the database level across all pages.

- Filename: `events-YYYY-MM-DD.csv`

**CSV generation pattern** (no external library needed):
```typescript
function toCsv(headers: string[], rows: string[][]): string {
  const escape = (val: string) => `"${val.replace(/"/g, '""')}"`
  return [headers.map(escape).join(','), ...rows.map(r => r.map(escape).join(','))].join('\n')
}
```
