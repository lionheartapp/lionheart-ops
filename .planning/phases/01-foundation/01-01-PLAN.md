3. `MaintenanceSpecialty` — same 8 values as MaintenanceCategory (categories map 1:1 to specialties per ROUTE-01)

4. `MaintenancePriority` with 4 values: `LOW`, `MEDIUM`, `HIGH`, `URGENT`

**Models to add:**

1. `MaintenanceTicket` (SCHEMA-01, SCHEMA-05, SCHEMA-07):
   - `id` String @id @default(cuid())
   - `organizationId` String
   - `ticketNumber` String (unique per org — format MT-0001; use `@@unique([organizationId, ticketNumber])`)
   - `title` String
   - `description` String?
   - `status` MaintenanceTicketStatus @default(BACKLOG)
   - `category` MaintenanceCategory
   - `specialty` MaintenanceSpecialty (auto-derived from category)
   - `priority` MaintenancePriority @default(MEDIUM)
   - `holdReason` HoldReason? (required when status = ON_HOLD)
   - `holdNote` String?
   - `completionNote` String?
   - `cancellationReason` String?
   - `scheduledDate` DateTime? (for SCHEDULED status)
   - `availabilityNote` String? (room access info from submitter)
   - `photos` String[] @default([]) (Supabase Storage URLs)
   - `completionPhotos` String[] @default([]) (QA requirement)
   - `aiAnalysis` Json? (cached AI diagnostic results — Phase 3)
   - `version` Int @default(1) (SCHEMA-05: offline sync prep)
   - `estimatedRepairCostUSD` Float? (LABOR-07 prep for FCI)
   - `submittedById` String (relation to User)
   - `assignedToId` String? (relation to User)
   - `buildingId` String? (relation to Building)
   - `areaId` String? (relation to Area)
   - `roomId` String? (relation to Room)
   - `schoolId` String? (relation to School — campus scoping)
   - `createdAt` DateTime @default(now())
   - `updatedAt` DateTime @updatedAt
   - `deletedAt` DateTime?
   - Relations: organization (Organization), submittedBy (User), assignedTo (User?), building (Building?), area (Area?), room (Room?), school (School?), activities (MaintenanceTicketActivity[]), technicianProfile via assignedTo
   - Indexes: `@@index([organizationId, status])`, `@@index([organizationId, assignedToId])`, `@@index([organizationId, submittedById])`, `@@index([organizationId, category])`, `@@index([organizationId, schoolId])`

   - `id` String @id @default(cuid())
   - `organizationId` String @unique
   - `lastTicketNumber` Int @default(0)
   - Relation: organization (Organization)

   - `metadata` Json? (extensible data)
