   Determine user role from the `usePermissions` hook. The permission-based logic should be:
   - `canManageMaintenance` = user has `maintenance:read:all` permission (Head or Admin)
   - `canClaimMaintenance` = user has `maintenance:claim` permission (Technician)
   - `canSubmitMaintenance` = user has `maintenance:submit` permission (everyone with module enabled)

   Based on these flags:
   - **Head/Admin** (`canManageMaintenance`):
     - "Facilities" link -> `/maintenance` (default tab: dashboard)
     - "Work Orders" link -> `/maintenance?tab=work-orders`
   - **Technician** (`canClaimMaintenance && !canManageMaintenance`):
     - "Facilities" link -> `/maintenance` (default tab: work-orders, showing assigned work)
     - "My Requests" link -> `/maintenance?tab=my-requests`
   - **Teacher/Staff** (`canSubmitMaintenance && !canClaimMaintenance && !canManageMaintenance`):
     - "Facilities" link -> `/maintenance` (default tab: my-requests)

   Each nav item follows the existing sidebar link pattern:
   ```tsx
   <PrefetchLink
     href="/maintenance"
     className={`flex items-center gap-3 rounded-lg px-3 py-2 text-sm transition-colors duration-200 ${
       pathname === '/maintenance' ? 'bg-primary-50 text-primary-700 font-medium' : 'text-gray-600 hover:bg-gray-100 hover:text-gray-900'
     }`}
   >
     <Wrench className="w-[18px] h-[18px]" />
     <span className="text-sm">Facilities</span>
   </PrefetchLink>
   ```

   For "Work Orders" use `ClipboardList` icon. For "My Requests" use `ClipboardList` icon.

6. **Add maintenance permissions to the usePermissions hook result.** Check `src/lib/hooks/usePermissions.ts` — if it returns specific boolean flags, add:
   - `canManageMaintenance` — checks `maintenance:read:all`
   - `canClaimMaintenance` — checks `maintenance:claim`
   - `canSubmitMaintenance` — checks `maintenance:submit`

   If the hook uses a generic pattern where permissions are checked dynamically, ensure the sidebar can check these permissions.

7. **The sidebar should NOT have a secondary panel for maintenance** (per CONTEXT decision: "No secondary sidebar panel — all sub-sections use in-page tabs like Athletics"). The nav items are simple links, not expandable panels.

8. **Maintenance nav items only appear when module is enabled.** Wrap the entire Support section in:
