  <behavior>
    Auth tests:
    - signAuthToken({userId:'u1', organizationId:'o1', email:'a@b.com'}) returns a string containing two dots (JWT format)
    - verifyAuthToken(validToken) returns the original claims object
    - verifyAuthToken('garbage') returns null
    - verifyAuthToken with a token signed with a different secret returns null

    Permission tests (mock rawPrisma):
    - can('user-1', 'tickets:read:all') returns true when user has matching permission in role
    - can('user-1', 'tickets:read:all') returns false when user has no matching permissions
    - can('user-1', 'anything') returns true when user has wildcard '*:*' permission
    - assertCan throws 'Insufficient permissions' when can() would return false
    - getUserPermissions caches results and returns cached on second call within TTL
    - clearPermissionCache causes next getUserPermissions call to re-fetch from DB

    Org-context tests:
    - runWithOrgContext('org-1', cb) makes getOrgContextId() return 'org-1' inside callback
    - getOrgContextId() throws 'Organization context is missing' when called outside runWithOrgContext
    - getOrgIdFromRequest throws 'Missing x-org-id header' when header is absent
    - getOrgIdFromRequest returns the header value when present
  </behavior>
