4. **Knowledge base service** (`src/lib/services/knowledgeBaseService.ts`):
   - `createArticle(data: { title, type, content, tags, assetId?, calculatorType?, isPublished?, createdById })`: Zod validate, create via prisma with org context
   - `getArticles(filters: { type?, tags?, assetId?, isPublished?, keyword? })`: Filter by type, keyword search on title+content+tags using Prisma string contains (case-insensitive). Sort by updatedAt desc.
   - `getArticleById(id)`: Full article with createdBy name and asset info
   - `updateArticle(id, data)`: Partial update with Zod validation
   - `deleteArticle(id)`: Soft-delete via prisma.knowledgeArticle.delete
   - `searchArticles(query: string, limit?: number)`: Search title, content, tags with case-insensitive contains. Returns top 10 matches by relevance (title match prioritized over content match).
   - `findRelevantArticles(category: string, title: string, keywords?: string[])`: Finds KB articles relevant to a maintenance ticket. Logic: (1) keyword search combining category + title words, (2) filter by isPublished=true, (3) return up to 5 results. No Gemini call — pure keyword matching is sufficient and fast.

5. **API routes** (all follow standard pattern: getOrgIdFromRequest → getUserContext → assertCan → runWithOrgContext):
   - `GET /api/maintenance/knowledge-base` — list with filters (?type=, ?keyword=, ?assetId=, ?tags=). Permission: KB_READ
   - `POST /api/maintenance/knowledge-base` — create article. Validate body with Zod. Permission: KB_CREATE
   - `GET /api/maintenance/knowledge-base/[id]` — single article. Permission: KB_READ
   - `PATCH /api/maintenance/knowledge-base/[id]` — update. Permission: KB_UPDATE
   - `DELETE /api/maintenance/knowledge-base/[id]` — soft-delete. Permission: KB_DELETE
   - `GET /api/maintenance/knowledge-base/search?q=&category=&limit=` — search endpoint used by AI panel. Permission: KB_READ. Calls `findRelevantArticles` when category param present, else `searchArticles`.
