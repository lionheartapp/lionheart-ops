# Phase 14: AI Assistant UX Upgrade - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning
**Source:** Product spec gap analysis + user requirements

<domain>
## Phase Boundary

Upgrade the AI assistant (Leo) from a text-only chat to a rich conversational UI with:
- Tappable button/choice responses (Typeform-style)
- Contextual suggestion chips after responses
- New tools: room availability, resource availability, weather forecast
- Smarter event creation with rich confirmation cards
- All within the existing 384px-wide chat panel using SSE streaming

**What already exists:**
- SSE streaming via Gemini generateContentStream (just shipped)
- "Leo" persona with dynamic system prompt
- 12 tools in assistant-tools.ts with permission gating
- ActionConfirmation component for write confirmations
- MessageList with streaming cursor + tool indicators
- ChatPanel with full streaming consumption
- weatherService.ts (Open-Meteo API — exists but no tool definition)
- calendarService.ts with room conflict detection (checkRoomConflict)
- EventResourceRequest model and resourceRequestService.ts
- Inventory system with InventoryItem model, checkout/checkin

**What does NOT change:**
- Gemini as AI provider (NOT switching to Claude/Anthropic)
- Existing tool execution functions (all execute* functions stay)
- Confirm route (/api/ai/assistant/confirm/route.ts)
- Voice input (useSpeechRecognition hook)
- ChatButton + AiGlow components
- Permission system, org-scoping, all existing patterns

</domain>

<decisions>
## Implementation Decisions

### Button/Choice UI
- When Leo asks a question with options, the AI response includes structured choices in a special format
- Choices render as tappable pill buttons below the assistant message
- Clicking a choice sends that text as the user's next message
- Choices disappear after one is selected (or after user types a custom response)
- Buttons use aurora gradient accent on hover (blue→indigo)
- Animated entrance with Framer Motion stagger

### Suggestion Chips
- After any assistant response, show 2-4 contextual follow-up suggestions
- Suggestions are generated by the AI as part of its response (structured format)
- Chips appear as a horizontal scrollable row below the last assistant message
- Clicking a chip sends it as the user's next message
- Chips disappear when a new message is sent
- Examples: after ticket stats → "Show by category", "Compare to last month", "Export"
- Examples: after event creation → "Add another event", "Check room availability"

### New Tools
- `check_room_availability` — check if a specific room is free for a date/time range
  - Uses existing calendarService room conflict detection
  - Returns: available (boolean), conflicting event details if booked
- `find_available_rooms` — find rooms matching criteria (capacity, campus)
  - Queries Room model with optional capacity/campus filters
  - Returns: list of available rooms for the requested time
- `check_resource_availability` — check inventory item availability
  - Queries InventoryItem for quantity available
  - Returns: available quantity, low-stock warning if applicable
- `get_weather_forecast` — weather for a specific date/location
  - Uses existing weatherService.ts (Open-Meteo API)
  - Returns: temperature, conditions, precipitation chance

### Rich Confirmation Cards
- When creating events/tickets, show a styled card instead of plain text
- Card includes: title, date/time, location, resources, approval chain preview
- Resource availability shown inline (✓ available, ⚠️ short, ✗ unavailable)
- "Edit" button on each field to modify before confirming
- Confirm/Cancel buttons at bottom of card

### SSE Event Format Extensions
- New event type: `choices` — `{"type":"choices","options":["Option A","Option B"]}`
- New event type: `suggestions` — `{"type":"suggestions","items":["Suggestion 1","Suggestion 2"]}`
- New event type: `rich_confirmation` — `{"type":"rich_confirmation","card":{...}}`
- These are parsed from the AI's structured response format

### System Prompt Updates
- Instruct Leo to include `[CHOICES: option1 | option2 | option3]` when asking questions
- Instruct Leo to include `[SUGGEST: suggestion1 | suggestion2]` after data responses
- The streaming route parses these markers and converts to SSE events
- This approach works with any LLM (no special function calling needed)

### Claude's Discretion
- Exact animation timing and easing for button/chip entrance
- Chip scroll behavior on overflow (horizontal scroll vs wrap)
- Whether rich confirmation card replaces ActionConfirmation or extends it
- Exact tool parameter schemas for new tools
- How to handle weather API failures gracefully

</decisions>

### AI Suggestion Examples
- After maintenance stats: "Show by category" | "Compare to last month" | "Top locations"
- After event creation: "Add another event" | "Check room availability" | "Invite attendees"
- After search results: "Show details" | "Create ticket for this" | "Search again"
- After ticket lookup: "Update status" | "Assign to someone" | "View similar tickets"

### Design Constraints
- Chat panel is 384px wide, 520px tall
- Buttons must be touch-friendly (min 44px tap target)
- Aurora gradient: `linear-gradient(90deg, #3B82F6 0%, #6366F1 100%)`
- Card style: `bg-white border border-gray-200 rounded-xl`
- Use Framer Motion for all animations (already in project)
- Glassmorphism classes available: ui-glass, ui-glass-hover

</specifics>
