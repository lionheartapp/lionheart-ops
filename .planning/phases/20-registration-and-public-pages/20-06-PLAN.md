**1. src/app/api/events/projects/[id]/share/route.ts** — Share config API.

- `GET` — Returns the share configuration for this EventProject. Requires `events:registration:manage` permission. Returns:
  - shareUrl: `${NEXT_PUBLIC_APP_URL}/events/${orgSlug}/${form.shareSlug}`
  - qrCodeSvg: generated server-side via `qrcode.toString(shareUrl, { type: 'svg', width: 300 })` — for embedding in flyers
  - qrCodeDataUrl: `qrcode.toDataURL(shareUrl, { width: 300 })` — for download
  - openAt, closeAt, maxCapacity, registeredCount, waitlistedCount
  - isOpen: computed from openAt/closeAt vs now
  Use the existing `qrcode` package already installed. If no registration form exists, return 404 with message "Set up registration first".

- `PUT` — Updates share settings. Body: `{ openAt?, closeAt?, brandingOverride?, maxCapacity?, waitlistEnabled? }`. Validates with Zod. Updates the RegistrationForm record. Requires `events:registration:manage`.

**2. src/app/api/events/projects/[id]/registrations/route.ts** — Registration list for staff.

- `GET` — Returns paginated list of registrations for this EventProject. Requires `events:registration:manage` permission. Uses standard org-context pattern.
  - Query params: `?status=REGISTERED|WAITLISTED|CANCELLED&page=1&limit=50&search=email|name`
  - Returns: registrations array (id, firstName, lastName, email, status, paymentStatus, submittedAt, photoUrl, grade), total count, capacity info (maxCapacity, registeredCount, waitlistedCount)
  - Does NOT include sensitiveData in list endpoint (separate medical endpoint)
  - Sort by submittedAt desc

**3. src/app/api/events/projects/[id]/registrations/[regId]/medical/route.ts** — FERPA-gated medical data.

- `GET` — Returns RegistrationSensitiveData for a specific registration.
  1. Requires `events:medical:read` permission via `assertCan(ctx.userId, PERMISSIONS.EVENTS_MEDICAL_READ)`.
  2. If permission denied, return 403 "Insufficient permissions to view medical data".
  3. Fetch RegistrationSensitiveData by registrationId.
  4. If not found, return 404.
  5. Return the sensitive data (allergies, medications, medicalNotes, emergencyName, emergencyPhone, emergencyRelationship).

All routes follow standard API pattern: getOrgIdFromRequest, getUserContext, assertCan, runWithOrgContext (for staff routes), try/catch/fail.
