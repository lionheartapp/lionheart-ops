**1. src/lib/services/registrationService.ts** — Core registration CRUD and capacity management.

Functions to implement:

- `createRegistrationForm(data: { organizationId, eventProjectId, title?, shareSlug, requiresPayment?, basePrice?, depositPercent?, maxCapacity?, waitlistEnabled?, requiresCoppaConsent?, openAt?, closeAt?, brandingOverride?, discountCodes? })` — Creates RegistrationForm with rawPrisma. Generates shareSlug from event title if not provided (slugify + random suffix for uniqueness). Returns created form.

- `updateRegistrationForm(formId: string, data: Partial<...>)` — Updates form config. Uses rawPrisma.

- `getRegistrationForm(eventProjectId: string)` — Returns form with sections and fields, sorted by sortOrder.

- `getRegistrationFormBySlug(shareSlug: string)` — Public lookup by slug. Returns form + event project (title, description, coverImageUrl, startsAt, endsAt) + organization (name, logoUrl, theme). Does NOT include sensitive data or payment secrets.

- `upsertFormSections(formId: string, sections: Array<{ id?, title, description?, sortOrder, fields: Array<{ id?, fieldType, fieldKey?, inputType, label, helpText?, placeholder?, required?, enabled?, options?, sortOrder }> }>)` — Deletes existing sections/fields for the form, then creates new ones. Uses rawPrisma.$transaction for atomicity.

- `submitRegistration(data: { shareSlug, firstName, lastName, email, phone?, grade?, photoUrl?, tshirtSize?, dietaryNeeds?, coppaConsentAt?, coppaConsentIp?, responses: Array<{ fieldId, value?, values?, fileUrl? }>, sensitiveData?: { allergies?, medications?, medicalNotes?, emergencyName?, emergencyPhone?, emergencyRelationship? } })` — Core registration flow:
  1. Look up form by shareSlug, verify open (openAt/closeAt check)
  2. Resolve organizationId and eventProjectId from form
  3. Check capacity: count REGISTERED registrations vs maxCapacity
  4. Create EventRegistration with status REGISTERED (if under capacity) or WAITLISTED (if at capacity and waitlistEnabled)
  5. Create RegistrationResponse rows for each response
  6. If sensitiveData provided, create RegistrationSensitiveData row
  7. Set submittedAt = now(), return registration with status
  CRITICAL: Use rawPrisma (parent has no org context). Set organizationId explicitly from the form's org.

- `cancelRegistration(registrationId: string)` — Sets status to CANCELLED, then calls `promoteFromWaitlist`.

- `promoteFromWaitlist(eventProjectId: string, organizationId: string)` — Atomic promotion per research pattern:
  Use rawPrisma.$transaction. Re-check capacity. Find oldest WAITLISTED. Update to REGISTERED with promotedAt. Return promoted registration (or null).

- `getRegistrations(eventProjectId: string, options?: { status?, includeDeleted? })` — List registrations. Does NOT include sensitiveData by default.

- `getRegistrationWithSensitiveData(registrationId: string)` — Returns registration with sensitiveData included. Caller must verify events:medical:read permission before calling.

**2. src/lib/services/registrationPaymentService.ts** — Stripe payment management.

```typescript
import Stripe from 'stripe'
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!)
```

Functions:
- `createPaymentIntent(data: { registrationId, amount: number, paymentType: 'FULL' | 'DEPOSIT' | 'BALANCE', discountCode?: string, eventProjectId, organizationId })` —
  1. If discountCode provided, call applyDiscountCode to compute discounted amount
  2. Create Stripe PaymentIntent with `automatic_payment_methods: { enabled: true }`, metadata: { registrationId, eventProjectId, organizationId, paymentType, discountCode }
  3. Create RegistrationPayment row with stripePaymentIntentId, amount, status 'requires_action', paymentType
  4. Return { clientSecret, paymentIntentId, amount }

- `applyDiscountCode(baseAmount: number, discountCode: string, formId: string)` — Looks up discount code in RegistrationForm.discountCodes JSON. Validates code exists, hasn't exceeded maxUses. Returns { discountedAmount, discountAmount }. Throws if code invalid.

- `handlePaymentSuccess(stripePaymentIntentId: string)` — Called from webhook. Upserts RegistrationPayment with status 'succeeded', paidAt. Updates EventRegistration.paymentStatus to DEPOSIT_PAID or PAID based on paymentType. Uses rawPrisma.

- `calculateAmount(form: RegistrationForm, paymentType: 'FULL' | 'DEPOSIT')` — Returns amount in cents. DEPOSIT = basePrice * depositPercent / 100. FULL = basePrice.

**1. src/app/api/events/projects/[id]/registration-config/route.ts** — Staff-facing CRUD for registration form configuration. Follows standard API route pattern with auth + org context.

- `GET` — Returns registration form + sections + fields for this EventProject. Requires `events:registration:manage` permission. Uses `getOrgIdFromRequest` + `getUserContext` + `assertCan`. Calls `registrationService.getRegistrationForm(eventProjectId)`.

- `POST` — Creates a new registration form for this EventProject. Validates with Zod: `{ title?, shareSlug?, requiresPayment?, basePrice?, depositPercent?, maxCapacity?, waitlistEnabled?, requiresCoppaConsent?, openAt?, closeAt?, brandingOverride?, discountCodes? }`. Generates shareSlug from event title if not provided. Requires `events:registration:manage`.

- `PUT` — Updates form config + sections/fields. Accepts `{ form: Partial<FormData>, sections: SectionData[] }`. Calls `updateRegistrationForm` + `upsertFormSections`. Requires `events:registration:manage`.

All routes follow the standard try/catch/fail pattern from CLAUDE.md.

**2. src/app/api/events/register/[eventSlug]/route.ts** — PUBLIC route (no auth). The `eventSlug` param is the RegistrationForm.shareSlug.

- `GET` — Returns public event info + form config + sections + fields for rendering the registration wizard. Calls `getRegistrationFormBySlug(shareSlug)`. Returns org branding (logo, primaryColor), event details (title, description, dates, coverImage), form sections with fields. Does NOT return capacity numbers or other internal data. Returns 404 if form not found or not within open/close dates.

- `POST` — Submits a registration. Validates Turnstile token FIRST via `verifyTurnstile(body.turnstileToken)`. If invalid, return 422 CAPTCHA_FAILED. Then validates body with Zod. Calls `submitRegistration(...)`. If registration requires payment, return `{ registration, requiresPayment: true }` so frontend knows to proceed to payment step. If no payment required, call `sendConfirmationEmail(registration.id)` and return `{ registration, requiresPayment: false }`. Apply rate limiting: 10 registrations per IP per hour using existing RateLimiter pattern.

**3. src/app/api/events/register/[eventSlug]/payment-intent/route.ts** — PUBLIC route (no auth).

- `POST` — Creates a Stripe PaymentIntent. Validates Turnstile token first. Body: `{ registrationId, paymentType: 'FULL' | 'DEPOSIT', discountCode? }`. Looks up registration and form, calculates amount via `calculateAmount`, calls `createPaymentIntent`. Returns `{ clientSecret, amount, currency }`. Rate limited: 5 per IP per hour.

CRITICAL: Both public routes must resolve organizationId from the event slug lookup (NOT from URL params). This prevents cross-tenant attacks per research Pitfall 1.
