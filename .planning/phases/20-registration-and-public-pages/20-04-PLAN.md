**1. src/components/registration/RegistrationWizard.tsx** — Multi-step registration form.

```typescript
'use client'
```

Props: `{ shareSlug: string, form: FormConfig, sections: FormSection[], orgBranding: { logoUrl?, primaryColor?, name } }`

State management: Single `formData` state object holding all field responses. Use `useReducer` for complex state. Steps derived from sections.sortOrder.

**Step flow:**
1. If `form.requiresCoppaConsent` — first step is COPPA consent interstitial: checkbox "I confirm I am the parent/legal guardian of the participant and consent to the collection of their personal information as described in this form." with consent timestamp capture (`coppaConsentAt = new Date().toISOString()`) and IP capture (via a lightweight server call or set on the server during submission). Must be checked to proceed. Note: `requiresCoppaConsent` is a staff-configured flag on the RegistrationForm, set in the form builder.
2. Each form section becomes a wizard step. Render fields based on inputType:
   - TEXT/NUMBER: `<input>` with label, helpText, placeholder, required indicator
   - DROPDOWN: `<select>` with options from field.options
   - CHECKBOX: Checkboxes for each option in field.options
   - DATE: `<input type="date">`
   - FILE: File upload component that:
     a. Shows a file picker button
     b. On file selection, POSTs to `/api/events/register/{shareSlug}/upload` with `{ fileName, contentType, registrationId }` to get a signed URL
     c. Uploads the file directly to Supabase using the signed URL
     d. Stores the returned `publicUrl` in the form data (for 'photo' fieldKey, this becomes the registration's `photoUrl`)
     e. Shows upload progress indicator and thumbnail preview for images
   - SIGNATURE: `<SignatureField>` component
   - COMMON fields with medical keys (allergies, medications, medical_notes, emergency_*) render with a small lock icon indicating FERPA protection
3. If `form.requiresPayment` — final step before submission is `<PaymentStep>`
4. Last step: Review summary showing all entered data, Turnstile widget, Submit button

**Progress indicator:**
- Top of wizard: horizontal step dots showing current position
- Step labels from section titles
- aria-label for accessibility: "Step {n} of {total}"
- Completed steps show checkmark, current step is highlighted with org primaryColor

**Navigation:**
- Back/Next buttons at bottom of each step
- Next validates required fields on current step before advancing
- On final submit: POST to `/api/events/register/{shareSlug}` with formData + turnstileToken + coppaConsentAt/coppaConsentIp (if applicable)
- Show loading spinner during submission
- On success (no payment): redirect to a thank-you page or show success message
- On success (with payment): PaymentStep handles payment flow, then submits

**Save draft:** On each "Next" click, optionally save partial data to server (PATCH with status DRAFT). This prevents data loss if parent closes browser. Use a debounced auto-save for long forms.
