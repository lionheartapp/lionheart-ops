**1. src/lib/services/registrationMagicLinkService.ts** — Magic link lifecycle.

```typescript
import { createHash, randomBytes } from 'crypto'
import { SignJWT, jwtVerify } from 'jose'
import { rawPrisma } from '@/lib/db'
import { RateLimiter } from '@/lib/rate-limit'
```

Create two rate limiters:
```typescript
const emailRateLimiter = new RateLimiter({ windowMs: 60 * 60 * 1000, max: 3 })  // 3 per email per hour
const ipRateLimiter = new RateLimiter({ windowMs: 60 * 60 * 1000, max: 10 })    // 10 per IP per hour
```

Functions:

- `checkRateLimit(email: string, ip: string)` — Check both rate limiters. Returns `{ allowed: boolean, retryAfterSec?: number }`. Key the email limiter on `email:${email.toLowerCase()}`, the IP limiter on `ip:${ip}`.

- `issueMagicLink(email: string, registrationId: string, organizationId: string)` —
  1. Generate raw token: `randomBytes(32).toString('hex')`
  2. Hash it: `createHash('sha256').update(rawToken).digest('hex')`
  3. Set expiresAt: now + 48 hours
  4. Create RegistrationMagicLink row with rawPrisma: { organizationId, email, registrationId, tokenHash, expiresAt }
  5. Build the link URL: `${process.env.NEXT_PUBLIC_APP_URL}/events/portal?token=${rawToken}`
  6. Send email via emailService with link, event name, org branding
  7. Return { success: true }

- `consumeMagicLink(rawToken: string)` —
  1. Hash the raw token
  2. Find the RegistrationMagicLink by tokenHash (rawPrisma)
  3. Validate: exists, not expired, usedAt is null
  4. Mark as used: update usedAt = now()
  5. Fetch the registration to get organizationId, email
  6. Sign a portal JWT using jose:
     ```typescript
     const secret = new TextEncoder().encode(process.env.AUTH_SECRET!)
     const portalToken = await new SignJWT({
       type: 'portal',
       registrationId: link.registrationId,
       organizationId: link.organizationId,
       email: link.email,
     })
       .setProtectedHeader({ alg: 'HS256' })
       .setExpirationTime('4h')
       .sign(secret)
     ```
  7. Return { portalToken, registrationId }

- `verifyPortalToken(token: string)` —
  1. Verify JWT signature with jose
  2. Check `type === 'portal'` claim
  3. Return payload: { registrationId, organizationId, email }
  4. Throw if invalid/expired
  CRITICAL: This is NOT a staff token. Do NOT call assertCan() on portal routes. Verify the portal token type explicitly.

**2. src/app/api/registration/magic-link/request/route.ts** — PUBLIC route, rate-limited.

- `POST` — Body: `{ email }`. Validates email with Zod.
  1. Check rate limit (both email and IP). If blocked, return 429 with retryAfterSec.
  2. Look up all EventRegistrations for this email (rawPrisma). If none found, still return 200 (do NOT leak whether email has registrations — security best practice).
  3. If registrations found, call `issueMagicLink` for each active registration.
  4. Return `{ ok: true, message: 'If you have registrations, a link has been sent to your email' }`.

**3. src/app/api/registration/magic-link/validate/route.ts** — PUBLIC route.

- `GET` — Query param: `?token=xxx`.
  1. Call `consumeMagicLink(token)`.
  2. On success: return `{ ok: true, portalToken, registrationId }`.
  3. On failure (expired/used/invalid): return 401 with error message.
  The client saves portalToken to localStorage and redirects to the portal page.
