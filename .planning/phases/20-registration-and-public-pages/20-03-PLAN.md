**1. src/lib/hooks/useRegistrationForm.ts** — TanStack Query hooks for registration form CRUD.

```typescript
'use client'
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
```

- `useRegistrationForm(eventProjectId: string)` — GET /api/events/projects/{id}/registration-config. Returns `{ data: { form, sections, fields }, isLoading, error }`. Query key: `['registration-form', eventProjectId]`.

- `useCreateRegistrationForm(eventProjectId: string)` — POST mutation. On success, invalidates query key.

- `useUpdateRegistrationForm(eventProjectId: string)` — PUT mutation. Sends `{ form: Partial<FormConfig>, sections: SectionData[] }`. On success, invalidates query key.

Define TypeScript interfaces at the top:
```typescript
interface FormConfig {
  id: string
  title: string | null
  requiresPayment: boolean
  basePrice: number | null
  depositPercent: number | null
  maxCapacity: number | null
  waitlistEnabled: boolean
  requiresCoppaConsent: boolean
  openAt: string | null
  closeAt: string | null
  shareSlug: string
  discountCodes: DiscountCode[] | null
}

interface DiscountCode {
  code: string
  percentOff: number | null
  amountOff: number | null
  maxUses: number | null
  usedCount: number
}

interface FormSection {
  id?: string
  title: string
  description: string | null
  sortOrder: number
  fields: FormField[]
}

interface FormField {
  id?: string
  fieldType: 'COMMON' | 'CUSTOM'
  fieldKey: string | null
  inputType: 'TEXT' | 'DROPDOWN' | 'CHECKBOX' | 'NUMBER' | 'DATE' | 'FILE' | 'SIGNATURE'
  label: string
  helpText: string | null
  placeholder: string | null
  required: boolean
  enabled: boolean
  options: Array<{ label: string; value: string }> | null
  sortOrder: number
}
```

Export all interfaces for use by other components.

**3. src/components/registration/FormBuilder.tsx** — Main orchestrator.

Props: `{ eventProjectId: string }`

State: Manages local copy of sections + fields (initialized from server data via useRegistrationForm). All edits are local until "Save" is clicked.

Layout:
- Header row: "Registration Form Builder" title + Save button (primary dark pill style)
- Two-column layout on lg:
  - Left (2/3): Section editors stacked vertically, "Add Section" button at bottom
  - Right (1/3): CommonFieldPicker + Form Settings panel

- CommonFieldPicker integration: When a common field is toggled ON, it gets added to the first section (or creates a "Participant Information" section if none exists). When toggled OFF, it's removed from whichever section it's in. Track enabled common field keys by scanning all fields with fieldType=COMMON.

- Form Settings panel (right column, below CommonFieldPicker):
  - requiresPayment toggle
  - If requiresPayment: basePrice input (dollars, convert to cents), depositPercent input
  - maxCapacity input
  - waitlistEnabled toggle
  - requiresCoppaConsent toggle
  - openAt / closeAt datetime inputs
  - Discount codes section: list of codes with add/edit/remove

- Save: Calls `useUpdateRegistrationForm` with `{ form: configChanges, sections: allSections }`. Show success toast via sonner. Show error toast on failure.
