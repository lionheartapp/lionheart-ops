### Integration Issues (3 items)

**1. COM-03 — Missing Prisma enum value (HIGH)**
`NOTIFICATION_RULES_RECALCULATED` string literal in `eventProjectService.ts:246` is not in `EventActivityType` enum. Prisma throws at runtime when event dates are updated and notification rules exist. Core dispatch works; only the activity log write fails.
**Fix:** Add enum value to schema.prisma + db:push.

**2. AI-05 — Orphaned generate-schedule endpoint (MEDIUM)**
`POST /api/events/ai/generate-schedule` exists but no UI component calls it. Schedule generation works through the create-from-description wizard flow. Staff cannot regenerate a schedule for an existing event.
**Fix:** Add "Regenerate with AI" button to EventScheduleTab.

**3. EVNT-04 — People tab placeholder (LOW)**
`EventPeopleTab.tsx` shows only static text. All participant data lives in the Registration tab. Design decision, not a missing feature.
**Fix:** Wire participant data or remove the tab.
