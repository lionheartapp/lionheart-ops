**src/lib/hooks/usePresence.ts:**
Combines Supabase Realtime with the DB-backed heartbeat:
- `usePresence(eventProjectId: string, userId: string, activeTab?: string)` —
  1. On mount: POST to /api/events/projects/${id}/presence with activeTab.
  2. Set up a setInterval (every 30s) to POST heartbeat.
  3. On unmount: DELETE to /api/events/projects/${id}/presence.
  4. Also subscribe to a Supabase Realtime channel `presence:${eventProjectId}`:
     - Use @supabase/supabase-js createClient (browser-side, anon key).
     - Channel: `event-presence-${eventProjectId}`.
     - Track: { userId, activeTab }.
     - On presence sync: update local state with list of active users.
  5. Returns: `{ activeUsers: { userId, name, avatarUrl, activeTab }[] }`.
  6. Fallback: if Supabase Realtime is not available (no NEXT_PUBLIC_SUPABASE_URL env var), fall back to polling GET /presence every 15s.

NOTE: The Supabase client needs NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY environment variables. If not set, gracefully degrade to polling.

Create a small utility: `src/lib/supabase-browser.ts` that exports a singleton browser Supabase client (if env vars are available) or null.
