**src/lib/services/eventAnnouncementService.ts:**

- `createAnnouncement(data: { eventProjectId, title, body, audience, targetGroupId?, createdById })` — creates EventAnnouncement with sentAt set to now(). After creation:
  1. Determines recipient list based on audience:
     - ALL: all REGISTERED participants for this event.
     - GROUP: participants in the specified group (via EventGroupAssignment where groupId=targetGroupId).
     - INCOMPLETE_DOCS: participants with at least one incomplete required document (via EventDocumentCompletion).
     - PAID_ONLY: participants with paymentStatus=PAID.
  2. Sends email to each recipient's registration email address using registrationEmailService (or a simple sendEmail call — fire-and-forget, non-blocking). Email contains: announcement title, body text, link to portal.
  3. Creates in-app Notification rows for any staff users tagged in the announcement (using notificationService if applicable — or skip this for now and note it's a Phase 22 enhancement).
  Returns the created announcement.

- `listAnnouncements(eventProjectId: string)` — returns all announcements for an event, newest first, with author info (name, avatar).
- `deleteAnnouncement(id: string)` — hard-delete.
- `getAnnouncementsForRegistration(registrationId: string)` — returns announcements visible to this participant based on audience targeting. Logic:
  - ALL: always included.
  - GROUP: included if participant is in the targeted group.
  - INCOMPLETE_DOCS: included if participant has any incomplete required document.
  - PAID_ONLY: included if registration paymentStatus=PAID.
  This is the public-facing query for the parent portal.

**src/lib/services/eventSurveyService.ts:**

- `createSurvey(data: { eventProjectId, formId, opensAt?, closesAt? })` — creates EventSurvey. The formId references a RegistrationForm created via the existing form builder (reuse pattern per CONTEXT decision).
- `updateSurvey(id: string, data: { status?, opensAt?, closesAt? })` — update survey settings. Status transitions: DRAFT→ACTIVE→CLOSED.
- `deleteSurvey(id: string)` — hard-delete.
- `listSurveys(eventProjectId: string)` — returns surveys with response count.
- `submitSurveyResponse(data: { surveyId, registrationId, responses: Record<string, unknown> })` — creates EventSurveyResponse. Validates survey is ACTIVE and not past closesAt. Enforces unique constraint (one response per registration per survey).
- `getSurveyResults(surveyId: string)` — returns all responses with participant info. For aggregation: count responses per field value for dropdowns/checkboxes, list text responses.

**src/lib/services/eventPresenceService.ts:**

Presence tracking uses a DB-backed heartbeat for audit + Supabase Realtime for live updates:
- `updatePresence(eventProjectId: string, userId: string, activeTab?: string)` — upserts EventPresenceSession with lastSeenAt=now(). Uses `prisma` (org-scoped).
- `getActiveUsers(eventProjectId: string)` — returns presence sessions where lastSeenAt > 2 minutes ago. Includes user info (name, avatar, activeTab).
- `removePresence(eventProjectId: string, userId: string)` — deletes the session (on tab close / navigate away).

Note: The actual real-time push will use Supabase Realtime channels in the UI (Plan 07). This service just provides the DB-backed persistence layer.

**1. `/api/events/projects/[id]/announcements/route.ts`:**
- GET: assertCan(EVENTS_ANNOUNCEMENTS_MANAGE). Returns listAnnouncements(id).
- POST: assertCan(EVENTS_ANNOUNCEMENTS_MANAGE). Zod validates { title, body, audience: enum, targetGroupId? }. Calls createAnnouncement with ctx.userId as createdById. Returns 201.
- DELETE: assertCan(EVENTS_ANNOUNCEMENTS_MANAGE). Requires { announcementId }. Calls deleteAnnouncement.

**2. `/api/events/projects/[id]/surveys/route.ts`:**
- GET: assertCan(EVENTS_SURVEYS_MANAGE). Returns listSurveys(id).
- POST: assertCan(EVENTS_SURVEYS_MANAGE). Zod validates { formId, opensAt?, closesAt? }. Returns 201.
- PUT: assertCan(EVENTS_SURVEYS_MANAGE). Zod validates { surveyId, status?, opensAt?, closesAt? }. Calls updateSurvey.
- DELETE: assertCan(EVENTS_SURVEYS_MANAGE). Requires { surveyId }.

**3. `/api/events/projects/[id]/surveys/[surveyId]/responses/route.ts`:**
- GET: assertCan(EVENTS_SURVEYS_MANAGE). Returns getSurveyResults(surveyId). Staff-only aggregated view.
- POST: Public route (no auth — parents submit via portal). Zod validates { registrationId, responses }. Calls submitSurveyResponse. Returns 201. NOTE: This route should verify the registrationId belongs to this event's eventProjectId to prevent cross-event submission.

**4. `/api/events/projects/[id]/presence/route.ts`:**
- GET: assertCan(any event permission — use EVENT_PROJECT_READ). Returns getActiveUsers(id).
- POST: assertCan(EVENT_PROJECT_READ). Body: { activeTab? }. Calls updatePresence(id, ctx.userId, activeTab).
- DELETE: assertCan(EVENT_PROJECT_READ). Calls removePresence(id, ctx.userId).

**5. `/api/registration/[id]/announcements/route.ts`:**
Public route (parent portal access — no auth, registration ID is the access token):
- GET: Fetches registration by ID using rawPrisma. If not found, return 404. Calls getAnnouncementsForRegistration(id). Returns announcements filtered by audience.
- Add this path to middleware public paths if not already covered by `/api/registration/*`.
