**src/lib/hooks/useEventGroups.ts:**
Create TanStack Query hooks:
- `useGroups(eventProjectId, type?)` — GET groups with assignments count.
- `useGroupAssignments(groupId)` — GET assignments + unassigned pool.
- `useAssignToGroup()` — POST mutation. Invalidates groups + assignments queries.
- `useRemoveFromGroup()` — DELETE mutation. Invalidates groups + assignments queries.
- `useAutoAssign()` — POST mutation to auto-assign endpoint.
- `useActivities(eventProjectId)` — GET activities with signup counts.
- `useActivitySignup()` — POST/DELETE mutations.
- `useDietaryMedicalReport(eventProjectId)` — GET dietary/medical aggregation.
- `useCreateGroup()`, `useUpdateGroup()`, `useDeleteGroup()` — group CRUD mutations.

**src/lib/event-pdf-utils.ts:**
Client-side PDF generation using jsPDF (follow the label-utils.ts pattern — accept JsPDF constructor to avoid SSR issues). All PDFs include: school logo (if available), event name, date, and clean table layouts.

Create 6 PDF generator functions:
1. `generateBusManifest(JsPDF, data: { eventName, date, logo?, groups: { name, passengers: { name, grade, medicalFlags }[] }[] })` — Portrait letter. Group name as header, table: #, Name, Grade, Medical Flags (red icon). Page break per bus.
2. `generateCabinRoster(JsPDF, data: { eventName, date, logo?, groups: { name, leader, participants: { name, grade, photoDataUrl? }[] }[] })` — Portrait letter. Group name + leader. Grid layout with small photo thumbnails (if available), name, grade.
3. `generateMedicalSummary(JsPDF, data: { eventName, date, logo?, participants: { name, allergies, medications, medicalNotes, emergencyName, emergencyPhone }[] })` — Portrait letter. Table: Name, Allergies, Medications, Notes, Emergency Contact. Red highlight for rows with medications.
4. `generateEmergencyContacts(JsPDF, data: { eventName, date, logo?, participants: { name, grade, emergencyName, emergencyPhone, emergencyRelation }[] })` — Portrait letter. Clean table: Participant, Grade, Contact Name, Phone, Relationship.
5. `generateActivityRoster(JsPDF, data: { eventName, date, logo?, activities: { name, time?, location?, participants: { name, grade }[] }[] })` — Portrait letter. Activity name header, time/location, table of participants.
6. `generateCheckInSheet(JsPDF, data: { eventName, date, logo?, participants: { name, grade, group? }[] })` — Portrait letter. Table: #, Name, Grade, Group, Check-In (empty checkbox column for manual check-in).

Each function creates a new jsPDF document and returns it (caller calls doc.save()). Use consistent font sizes: title 16pt, section headers 12pt, table text 10pt. Add page numbers at bottom. The context doc says "PDFs should feel professional — the kind of thing a school would hand to a bus driver or camp counselor."
