**Prisma schema — add EventTemplate model:**
- id (cuid), organizationId, name (String), description (String?)
- sourceEventProjectId (String?) — the event this template was saved from
- templateData (Json) — serialized structure: { schedule: ScheduleBlock[], budgetCategories: string[], taskTemplates: TaskTemplate[], documentTypes: string[], groupStructure: GroupTemplate[], notificationRules: NotificationRuleTemplate[] }
- eventType (String?) — categorization hint (e.g., "camp", "field_trip", "retreat")
- expectedAttendance (Int?)
- durationDays (Int?)
- isMultiDay (Boolean, default false)
- usageCount (Int, default 0) — how many times used
- lastUsedAt (DateTime?)
- createdById (FK to User), createdAt, updatedAt
- Relations: organization, sourceEventProject (EventProject?), createdBy (User)
- @@index([organizationId]), @@index([organizationId, eventType])

Add relation to EventProject: `templates EventTemplate[]` (one event can be source for multiple templates).

In src/lib/db/index.ts: Add 'EventTemplate' to orgScopedModels.

In src/lib/permissions.ts: Add:
- EVENTS_TEMPLATES_MANAGE: 'events:templates:manage'
Add to super-admin and admin role arrays. Add EVENTS_TEMPLATES_MANAGE to member role array (members can use templates too).

**Create src/lib/types/event-template.ts:**
- ScheduleBlockTemplate: { dayOffset: number, startTime: string, endTime: string, title: string, type: string, location?: string }
- TaskTemplate: { title: string, category?: string, priority?: string }
- GroupTemplate: { name: string, type: string, capacity?: number }
- NotificationRuleTemplate: { triggerType: string, offsetDays?: number, conditionType?: string, label: string, subject: string }
- TemplateData interface combining all above arrays
- CreateTemplateInput Zod schema: { name, description?, eventType? }

**Create src/lib/types/event-ai.ts:**
- AIEventSuggestion: { title, description, startsAt, endsAt, isMultiDay, expectedAttendance, locationText, suggestedDocs: string[], suggestedTasks: TaskTemplate[], budgetEstimate: { category: string, estimatedMin: number, estimatedMax: number }[], scheduleBlocks: ScheduleBlockTemplate[] }
- AIScheduleSuggestion: { blocks: ScheduleBlockTemplate[], reasoning: string }
- AIBudgetEstimate: { categories: { name: string, estimatedMin: number, estimatedMax: number }[], totalMin: number, totalMax: number, reasoning: string }
- AIStatusSummary: { summary: string, completionPercent: number, atRisk: string[], nextSteps: string[] }

**Create src/lib/services/eventTemplateService.ts:**

- `getTemplates(opts?: { eventType?: string })` — Returns all templates for org, optional filter by eventType, sorted by usageCount desc.

- `getTemplate(templateId: string)` — Returns single template with templateData.

- `saveAsTemplate(eventProjectId: string, input: CreateTemplateInput, userId: string)` — Reads EventProject with its schedule blocks, tasks, budget categories, document requirements, groups, notification rules. Strips all dates, participant data, and IDs. Serializes structure into templateData JSON. Creates EventTemplate. If event has EventRegistration records, adds expectedAttendance from count.

- `createFromTemplate(templateId: string, overrides: { title: string, startsAt: string, endsAt: string, locationText?: string }, userId: string)` — Creates new EventProject from template. Applies date offsets from template schedule blocks relative to new startsAt. Increments template usageCount and lastUsedAt. Returns the new EventProject ID.

- `deleteTemplate(templateId: string)` — Hard deletes template.

**Create src/lib/services/ai/eventAIService.ts:**

Uses GoogleGenAI client (same pattern as gemini.service.ts). All functions gracefully degrade if GEMINI_API_KEY not set (return null).

- `generateEventFromDescription(description: string, orgContext?: { existingEvents?: string[] })` — Sends description to Gemini with structured prompt requesting JSON output matching AIEventSuggestion. Includes school operations context (types of events, required docs like permission slips, budgeting). Returns parsed AIEventSuggestion or null.

- `generateSchedule(params: { eventType: string, durationDays: number, expectedAttendance: number, description?: string })` — Generates multi-day schedule with appropriate blocks. Returns AIScheduleSuggestion or null.

- `estimateBudget(params: { eventType: string, durationDays: number, expectedAttendance: number, location?: string })` — Estimates budget ranges per category. Returns AIBudgetEstimate or null.

- `generateStatusSummary(eventData: { title: string, status: string, registrationCount: number, totalTasks: number, completedTasks: number, documentCompletionPercent: number, budgetUtilization: number, daysUntilEvent: number })` — Generates natural language status summary. Returns AIStatusSummary or null.

- `generateNotificationDraft(params: { eventTitle: string, eventDate: string, triggerType: string, targetAudience: string, context?: string })` — Generates subject + body for a notification. Returns { subject: string, body: string } or null.

- `enhanceTemplateForReuse(templateData: TemplateData, newParams: { startsAt: string, endsAt: string }, sourceEventLessons?: string[])` — Adjusts template dates, suggests budget updates, surfaces lessons learned from source event's activity log. Returns enhanced TemplateData.

- `analyzeFeedback(surveyResponses: { question: string, answers: string[] }[])` — Analyzes post-event survey responses and returns { themes: { theme: string, sentiment: string, count: number }[], summary: string, actionItems: string[] }.

All Gemini prompts request JSON output. Parse with regex match for `{...}` or `[...]` patterns to handle markdown code block wrapping (consistent with existing AI pattern).

Create 5 API route files following standard pattern:

1. **src/app/api/events/templates/route.ts**
   - GET: assertCan(EVENTS_TEMPLATES_MANAGE). Optional query param ?eventType=. Calls getTemplates. Returns template list.
   - POST: assertCan(EVENTS_TEMPLATES_MANAGE). Body: { eventProjectId, name, description?, eventType? }. Calls saveAsTemplate. Returns 201 with template.

2. **src/app/api/events/templates/[id]/route.ts**
   - GET: assertCan(EVENTS_TEMPLATES_MANAGE). Calls getTemplate(id). Returns template with full data.
   - POST: assertCan(EVENTS_CREATE). Body: { title, startsAt, endsAt, locationText? }. Calls createFromTemplate. Returns 201 with new eventProjectId.
   - DELETE: assertCan(EVENTS_TEMPLATES_MANAGE). Calls deleteTemplate. Returns ok(null).

3. **src/app/api/events/ai/create-from-description/route.ts**
   - POST: assertCan(EVENTS_CREATE). Body: { description: string }. Calls generateEventFromDescription. Returns AIEventSuggestion. If AI not available, returns fail('AI_UNAVAILABLE', 'Gemini API key not configured') with 503.

4. **src/app/api/events/ai/generate-schedule/route.ts**
   - POST: assertCan(EVENTS_CREATE). Body: { eventType, durationDays, expectedAttendance, description? }. Calls generateSchedule. Returns AIScheduleSuggestion.

5. **src/app/api/events/ai/generate-summary/route.ts**
   - POST: assertCan(EVENTS_READ). Body: { eventProjectId }. Loads event data (registration count, task completion, document completion, budget utilization, days until event). Calls generateStatusSummary. Returns AIStatusSummary. Uses two-phase pattern: if called with ?skipAI=true, returns just the raw metrics without AI summary.

All routes use standard error handling. AI endpoints return 503 with clear message when Gemini is not configured (not a 500 — it's expected in development).
