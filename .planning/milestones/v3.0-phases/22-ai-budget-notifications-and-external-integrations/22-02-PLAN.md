**Create src/lib/services/notificationOrchestrationService.ts:**

Core functions:

- `getRules(eventProjectId: string)` — Returns all rules for event, sorted by scheduledAt (nulls last), with approvedBy user name.

- `createRule(eventProjectId: string, input: NotificationRuleInput, userId: string)` — Creates rule. For DATE_BASED: computes scheduledAt from event's startsAt + offsetDays. Status starts as DRAFT.

- `updateRule(ruleId: string, input: Partial<NotificationRuleInput>)` — Updates rule fields. Only allowed when status is DRAFT or PENDING_APPROVAL. If offsetDays changes, recomputes scheduledAt.

- `deleteRule(ruleId: string)` — Hard deletes rule. Only allowed when status is DRAFT.

- `submitForApproval(ruleId: string)` — Changes status from DRAFT to PENDING_APPROVAL.

- `approveRule(ruleId: string, userId: string)` — Changes status to APPROVED. Sets approvedById and approvedAt. For DATE_BASED rules, scheduledAt must be in the future.

- `cancelRule(ruleId: string)` — Changes status to CANCELLED.

- `recalculateRulesForEvent(eventProjectId: string)` — Called when event is rescheduled. Finds the EventProject's new startsAt. For all DATE_BASED rules with status DRAFT, PENDING_APPROVAL, or APPROVED: recomputes scheduledAt = startsAt + offsetDays. Returns array of { ruleId, label, oldScheduledAt, newScheduledAt } for summary display.

- `dispatchPendingNotifications()` — Called by cron. Finds all rules where status=APPROVED AND scheduledAt <= now() AND sentAt IS NULL. For each rule:
  1. Resolve target audience to user IDs (all staff, registered participants, specific group, etc.)
  2. For in-app: call createBulkNotifications from notificationService.ts
  3. For email: send via emailService (fire-and-forget)
  4. Update rule: sentAt = now(), status = SENT
  5. Create EventNotificationLog entry
  Uses rawPrisma for cross-org cron context.

- `resolveAudience(eventProjectId: string, targetAudience: string)` — Helper that resolves audience string to array of { userId, email, name }. "all" = all registrants; "registered" = confirmed only; "group:{id}" = members of specific group; "incomplete_docs" = registrants with incomplete document requirements; "unpaid" = registrants with paymentStatus != PAID.
