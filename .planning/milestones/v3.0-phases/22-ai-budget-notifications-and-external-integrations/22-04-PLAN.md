Create src/lib/hooks/useBudget.ts with TanStack Query hooks:

- `useBudgetData(eventProjectId: string)` — Fetches GET /api/events/projects/{id}/budget. Returns { categories, lineItems, isLoading, error }. queryKey: ['budget', eventProjectId].

- `useBudgetRevenue(eventProjectId: string)` — Fetches GET /api/events/projects/{id}/budget/revenue?sync=true (syncs Stripe data on load). Returns { revenue, isLoading }. queryKey: ['budget-revenue', eventProjectId].

- `useBudgetReport(eventProjectId: string)` — Fetches GET /api/events/projects/{id}/budget/report. Returns full BudgetReportData. queryKey: ['budget-report', eventProjectId].

- `useBudgetMutations(eventProjectId: string)` — Returns mutation functions:
  - addLineItem: POST to /budget, invalidates budget + report queries
  - updateLineItem: PATCH to /budget/{lineId}, invalidates budget + report queries
  - deleteLineItem: DELETE to /budget/{lineId}, uses optimistic removal from cache
  - addRevenue: POST to /budget/revenue, invalidates revenue + report queries
  - updateRevenue: PATCH to /budget/revenue, invalidates revenue + report
  - deleteRevenue: DELETE to /budget/revenue, optimistic removal
  - addCategory: POST to /budget/categories (add this to the budget route as needed, or include in the existing POST route as a type discriminator)

All mutations show success/error via the project's useToast hook (not sonner). Optimistic updates follow the Phase 21 cancelQueries + setQueryData pattern.
