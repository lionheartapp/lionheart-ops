<task type="auto">
  <name>Task 2: Budget service layer with CRUD, revenue tracking, and report aggregation</name>
  <files>src/lib/services/budgetService.ts</files>
  <action>
Create src/lib/services/budgetService.ts with these functions:

**Category management:**
- `initializeCategories(eventProjectId: string)` — Creates preset categories from BUDGET_CATEGORY_PRESETS with isPreset=true and sequential sortOrder. Idempotent — skips if categories already exist for this event.
- `getCategories(eventProjectId: string)` — Returns all categories sorted by sortOrder.
- `createCategory(eventProjectId: string, name: string)` — Creates custom category with isPreset=false, sortOrder = max existing + 1.
- `updateCategory(categoryId: string, data: { name?: string, sortOrder?: number })` — Updates name or sort order.
- `deleteCategory(categoryId: string)` — Deletes category. Fails with error if category has line items (referential integrity).

**Line item CRUD:**
- `getLineItems(eventProjectId: string, opts?: { categoryId?: string })` — Returns line items with category relation, optional filter by category.
- `createLineItem(eventProjectId: string, input: BudgetLineItemInput, userId: string)` — Creates line item. Validates categoryId belongs to same eventProject.
- `updateLineItem(lineId: string, input: Partial<BudgetLineItemInput>)` — Updates line item fields.
- `deleteLineItem(lineId: string)` — Hard deletes line item.

**Revenue:**
- `getRevenue(eventProjectId: string)` — Returns all revenue entries sorted by receivedDate desc.
- `createRevenue(eventProjectId: string, input: BudgetRevenueInput, userId: string)` — Creates revenue entry.
- `updateRevenue(revenueId: string, input: Partial<BudgetRevenueInput>)` — Updates revenue entry.
- `deleteRevenue(revenueId: string)` — Hard deletes revenue entry.
- `syncRegistrationRevenue(eventProjectId: string)` — Queries EventRegistration records with status CONFIRMED and paymentStatus PAID for this event, sums payment amounts, upserts a single BudgetRevenue row with source=REGISTRATION_FEE and isAutoPopulated=true. Uses description "Registration fee revenue (auto-synced)".

**Reporting:**
- `getBudgetReport(eventProjectId: string): Promise<BudgetReportData>` — Aggregates:
  - Per-category totals (sum budgeted, sum actual, count of line items)
  - Grand total budgeted (sum all budgetedAmount)
  - Grand total actual (sum all actualAmount where not null)
  - Total revenue (sum all BudgetRevenue.amount)
  - Net position (totalRevenue - totalActual)
  - Registration count (count EventRegistration where status != CANCELLED)
  - Per-participant cost (totalActual / registrationCount, or 0 if no registrations)
  All Decimal values converted to number for JSON serialization.

Use `prisma` (org-scoped) for all queries — these run inside runWithOrgContext. Use `(prisma as any).budgetLineItem` cast pattern consistent with Phase 21 decision for newer org-scoped models.

Import logger for error handling. All functions are async and throw on validation errors (caller handles response).
