**Prisma schema additions:**

1. **IntegrationCredential** — Stores OAuth tokens and API keys for integrations
   - id (cuid), organizationId, userId (String?) — null for org-level (PCO, Twilio), user-specific for Google Calendar
   - provider (String) — "planning_center", "google_calendar", "twilio"
   - accessToken (String?), refreshToken (String?), tokenExpiresAt (DateTime?)
   - config (Json?) — provider-specific config (e.g., Twilio phone number, PCO org name)
   - isActive (Boolean, default true)
   - lastSyncAt (DateTime?)
   - createdAt, updatedAt
   - Relations: organization, user (User?)
   - @@unique([organizationId, provider, userId]) — one credential per provider per user (or per org if userId null)
   - @@index([organizationId, provider])

2. **IntegrationSyncLog** — Audit trail of sync operations
   - id (cuid), organizationId, credentialId (FK to IntegrationCredential)
   - provider (String), action (String) — "sync_people", "push_event", "send_sms"
   - status (String) — "success", "partial", "failed"
   - recordsProcessed (Int, default 0), recordsFailed (Int, default 0)
   - errorMessage (String?), metadata (Json?)
   - createdAt
   - Relations: organization, credential
   - @@index([organizationId, provider])

Add to orgScopedModels: 'IntegrationCredential', 'IntegrationSyncLog'.

**Permissions:**
- INTEGRATIONS_MANAGE: 'integrations:manage' — admin-only, manage org-level integrations
- INTEGRATIONS_GOOGLE_CALENDAR: 'integrations:google-calendar' — per-user Google Calendar connect
Add INTEGRATIONS_MANAGE to super-admin and admin. Add INTEGRATIONS_GOOGLE_CALENDAR to admin and member.

**Create src/lib/types/integrations.ts:**
- IntegrationProvider: 'planning_center' | 'google_calendar' | 'twilio'

- TwilioConfigInput Zod schema: { accountSid, authToken, phoneNumber }
- SMSInput Zod schema: { to: string, body: string }

**Create src/lib/services/integrations/planningCenterService.ts:**
- `getAuthUrl(organizationId: string)` — Returns PCO OAuth authorization URL. Requires PCO_APP_ID and PCO_SECRET env vars. If not set, returns null.
- `handleCallback(organizationId: string, code: string)` — Exchanges code for tokens, stores in IntegrationCredential.
- `syncPeople(organizationId: string)` — Fetches people from PCO API (/people/v2/people). Matches by email. For matched users: updates name, phone. For unmatched: logs for manual review. Handles pagination. Creates IntegrationSyncLog.
- `syncServices(organizationId: string)` — Fetches service types and plans from PCO Services API. Maps to EventProjects or logs for review. Creates IntegrationSyncLog.
- `pushCheckIns(organizationId: string, eventProjectId: string)` — Pushes check-in data from EventCheckIn records to PCO Check-ins API. One-way push.
- `isAvailable()` — Returns boolean: true if PCO_APP_ID and PCO_SECRET are set.
All functions check isAvailable() first and gracefully return null/empty if not configured.

**Create src/lib/services/integrations/googleCalendarService.ts:**
- `getAuthUrl(userId: string)` — Returns Google OAuth URL with calendar.events scope. Requires GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET. Returns null if not set.
- `handleCallback(userId: string, organizationId: string, code: string)` — Exchanges code for tokens, stores per-user IntegrationCredential.
- `refreshTokenIfNeeded(credentialId: string)` — Checks tokenExpiresAt, refreshes if < 5 min remaining. Uses google-auth-library or direct HTTP.
- `syncEventToCalendar(userId: string, eventProject: EventProject)` — Creates or updates Google Calendar event. Maps EventProject fields to Google Calendar event format. Stores Google event ID in metadata.
- `removeEventFromCalendar(userId: string, googleEventId: string)` — Deletes event from user's Google Calendar.
- `isAvailable()` — Returns boolean: true if GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET are set.
Use googleapis npm package (already used by googleAdminService). All functions gracefully degrade.

**Create src/lib/services/integrations/twilioService.ts:**
- `sendSMS(organizationId: string, to: string, body: string)` — Sends SMS via Twilio. Reads credentials from IntegrationCredential for org. Creates IntegrationSyncLog. Returns { sid, status } or null if not configured.
- `sendBulkSMS(organizationId: string, recipients: { to: string, body: string }[])` — Sends to multiple recipients with rate limiting (max 1 per second). Uses Promise.allSettled. Creates summary log.
- `getDeliveryStatus(organizationId: string, messageSid: string)` — Checks Twilio message status.
- `isAvailable(organizationId: string)` — Checks if org has active Twilio IntegrationCredential.
Use twilio npm package. All functions gracefully degrade.

**Create API routes:**

1. **planning-center/auth/route.ts** — GET: returns auth URL. POST: handles callback code exchange.
2. **planning-center/sync/route.ts** — POST: body { action: 'people' | 'services' | 'checkins', eventProjectId? }. Triggers sync.
3. **google-calendar/auth/route.ts** — GET: returns per-user auth URL.
4. **google-calendar/callback/route.ts** — GET: handles OAuth callback, stores tokens, redirects to settings.
5. **google-calendar/sync/route.ts** — POST: body { eventProjectId }. Syncs specific event to user's calendar.
6. **twilio/config/route.ts** — GET: returns current Twilio config status. POST: saves Twilio credentials (account SID, auth token, phone number) to IntegrationCredential. DELETE: removes Twilio config.
7. **twilio/send/route.ts** — POST: body { to, body } or { eventProjectId, message }. Sends SMS.

All routes check isAvailable() and return 503 with clear message if integration not configured.

Run `npm run db:push`. Install `twilio` and `googleapis` packages if not present.
